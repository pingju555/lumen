## 二次排查：config.json 反复迁移导致种子永生（同次会话）

## 真正根因：ParseDoc 大小写不匹配（同次会话，终极修复）
新建 profile 永远带默认部件的**终极根因**：`Save` 用 `JsonSerializer.Serialize<Document>(doc, Opt)` 输出 **PascalCase** 键（`Pages/Atoms/Type`），但旧 `ParseDoc` 用 `JsonDocument` 手动读**小写**键（`"pages"/"atoms"`），`TryGetProperty` 大小写敏感 → 永远匹配不到 → 所有 profile 加载为空 → 每次启动/切档都走 `SeedDefaultProject()` 兜底 → 默认 4 部件永生。这也意味着任何靠加载显示的 profile（含手册）不修此 bug 就显示不出来。
修复：`ParseDoc` 改用 `JsonSerializer.Deserialize<Document>(json, Opt)` 强类型反序列化（`Opt.PropertyNameCaseInsensitive=true` → PascalCase/camelCase 通吃）；新增 `PageFromDto`/`PresetFromDto`/`BgFromDto` 做 DTO→领域转换；旧 `screens` schema 保留为 fallback。
教训：**序列化与反序列化必须走同一套键名约定**；手动 `JsonDocument` 读键极易与 `JsonSerializer` 输出的 PascalCase 脱节。优先强类型反序列化 + `PropertyNameCaseInsensitive`。
