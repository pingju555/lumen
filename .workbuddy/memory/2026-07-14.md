## 22:47 关键根因：MA_NOACTIVATE 导致最小化应用依次弹出
- 用户最终澄清真实问题：**应用程序在最小化状态下会自己"一个个冒出来"（被自动还原）**，不是"应用盖住覆盖层"。之前选的"恢复时盖住覆盖层"是误判。
- 根因：`WM_MOUSEACTIVATE -> MA_NOACTIVATE`（22:31 加的"点击不激活"技巧）。窗口永远不可激活 → Windows 找不到可激活 foreground → 逐个还原最小化窗口当 foreground，表现为"依次出现"。该 MA_NOACTIVATE 在 HWND_BOTTOM 版本里就已埋着，所以纯回退到旧状态**不会**消除弹窗。
- 修复：删除 WndProc 里的 `WM_MOUSEACTIVATE -> MA_NOACTIVATE` 分支（窗口恢复可激活）。保留 `HWND_TOPMOST`（应用不盖覆盖层）。还原 `GetDesktopRectExcludingTaskbar` 的 autohide 早返回（铺满屏、任务bar浮在上面）→ 去掉缩减条。
- 行为：覆盖层始终置顶可见；最小化应用不再自动弹出；任务栏无死区条；右键菜单正常（窗口已置顶，激活不改变层级）。应用在被覆盖时不可直接点（覆盖层为不透明置顶），需用 `Ctrl+Alt+O` 隐藏覆盖层再操作。若需"应用可见可点 + 覆盖层在上方"，走透明背景 + 点击穿透(WS_EX_TRANSPARENT / WM_NCHITTEST HTTRANSPARENT) 方案，单独排期。
- 构建 0 警告 0 错误。
- **可复用经验**：WPF 桌面 widget 想做"点击不抢焦点"不能用 MA_NOACTIVATE（会引发最小化窗口自动还原循环）；应改用置顶 + 选择性点击穿透(WM_NCHITTEST 对背景返回 HTTRANSPARENT)，且需配 AllowsTransparency 透明背景。

## 22:51 回退 HWND_TOPMOST → HWND_BOTTOM（解决"设置被覆盖 + 任务栏出不来"）
- 用户反馈：22:47 改的 `HWND_TOPMOST` 置顶方案引发两个新问题：① 设置弹窗被覆盖；② 自动隐藏任务栏唤不出（置顶铺满屏盖住任务栏触发区）。

  - `SettingsWindow.xaml` 加 `Topmost="True"`，确保设置窗口始终在覆盖层（及任何应用）之上、不被遮。
  - 注释清理：`GetDesktopRectExcludingTaskbar`（autohide 铺满屏、taskbar 浮在 HWND_BOTTOM 之上）、`SetTopMost`（标记未使用、保留备用）。
  - `MA_NOACTIVATE` 已在 22:47 删除，本版保留（避免最小化窗口自动还原）。
- 行为：覆盖层=桌面挂层（桌面之上、所有应用与任务栏之下）→ 应用永远可见、自动隐藏任务栏可唤出、右键菜单正常且不被覆盖、设置窗口不被覆盖、最小化应用不再自动弹出。要操作应用 → `Ctrl+Alt+O` 隐藏覆盖层。
- 构建 0 警告 0 错误。
