## 0. 体系总览（当前真实结构）

Lumen 不是 KLWP 那种「三栏常驻 + Overlay」单体编辑器，而是**主覆盖层 + 多个独立浮窗**的分散式结构：

```
┌─ LumenWindow（主覆盖层，常驻）
│   ├─ 桌面模式：盖桌面图标/壁纸，拦截输入，静态渲染
│   └─ 编辑模式(EditMode)：显网格/标尺/HUD，打开以下浮窗
│        ├─ TreeWindow     （原子树，独立 Window）
│        ├─ PropWindow     （属性编辑，内嵌 PropertyEditorPanel）
│        └─ GvManagerPanel （全局变量，CreateEditorWindow 承载）
└─ 独立管理窗（按需，不依赖编辑态）
    ├─ SettingsWindow      （程序设置）
    ├─ PageGridBgWindow    （页面/网格/背景/场景预设）
    ├─ ProfileWindow       （配置档管理）
    ├─ ColorPickerWindow   （取色器，被多处调用）
    ├─ FilePickerWindow    （文件选择）
    └─ InputBox            （轻量输入，取名用）
```

**关键差异点**：KLWP 的「项目列表 + 属性编辑器 + 工具栏」三栏是**同一个 Window 内的常驻面板**；Lumen 目前是**主窗 + 多个独立 Window 浮窗**并存，靠 `LumenWindow.ShowEditorWindows()/CloseEditorWindows()` 协同开关。

## 2. 属性编辑器（PropertyEditorPanel.xaml.cs）

### 2.1 分组结构 —— 已实现 Tab，但分类法不同
- `BuildTabs()`(`xaml.cs:102`)：依据原子 `EditTabs()` 生成 `TabControl`。已知 Tab 键 `KnownTabKeys = {content, style, layout, animation, interaction, flow}`(`xaml.cs:100`)。
- 字段经 `BuildFieldRow()`(`xaml.cs:155`) 按 `EditField.Tab` 归类；`interaction`/`flow` 为特殊页（`BuildInteractionBlock`/`BuildFlowBlock`）；自定义 Tab 经 `ICustomTabProvider.BuildCustomTab` 注入（如 ComponentAtom）。
- **没有** KLWP 的「形状/油漆/图层/位置/动画/触摸」六分类。当前是 `content/style/layout/animation/interaction/flow`，语义接近但名称与范围不同，**缺独立「图层」Tab**（Z 序/混合在右键菜单，无混合模式）。

### 2.2 控件类型（BuildValueEditor，xaml.cs:871，按 EditKind 分派）
| EditKind | 控件 | 现状 |
|----------|------|------|
| Text | `TextBox` | 裸文本框 |
| Choice | `ComboBox` | `Tag`=规范值，`ChoiceLocPrefix` 本地化 |
| Color | 色块+Hex+「调色板」按钮 | 调 `ColorPickerWindow` |
| File | `TextBox`+浏览 | `FilePickerWindow` |
| Number | `TextBox` | 裸文本框 |
| Bool | `CheckBox` | — |
| Slider | `Slider`+联动 `TextBox` | 连续值 |

**无步进器控件**（设计 P0 的 ◀▶ 大跳 + −/+ 微调）。全代码库无 `RepeatButton`/`Stepper`，仅 ProgressAtom 注释提到。

### 2.3 三态值/公式/变量
- 每个字段行是「标签 + 三态输入」：`MakeModeToggle()`(`xaml.cs:816`) 给 `值/公式/变量` 三个小按钮切换 `PropMode`（`Static/Formula/Global`）。
- **与 KLWP 差异**：KLWP 是每行一个独立 `□` 公式绑定图标（空心=未绑，实心红=已绑）。Lumen 是「值/公式/变量」三段切换，**无「该字段已绑公式」的专属方块视觉提示**。

### 2.4 公式绑定入口（字段级）
- `BuildFormulaHost()`(`xaml.cs:1032`)：纯 `TextBox`（原内联 `FormulaTextBox` 已移除，注释 `xaml.cs:1035`，为避免 Popup 焦点问题）+ `✓/✗` 语法状态行 `UpdateFormulaStatus` + 结果预览 `UpdateFormulaPreview` + 右上「📖 函数…」按钮 `OpenFunctionPicker`(`xaml.cs:1097`)。

**与设计差距**：字段级预览已有（绿=结果/红=失败）；缺 KLWP 那种弹窗级「实时预览区 + 函数分类网格卡片」整合视图。

## 3. 公式引擎与函数目录

### 3.1 引擎（Formula/ 目录）
`Lexer`/`Parser`/`Ast`/`Value`/`FormulaEngine`/`EvalContext` + `FunctionRegistry`（~19 函数）。Provider：`si`(PDH CPU/内存/网速/磁盘)、`bi`(内置)、`gv`(全局变量)、`mi`/`mu`(SMTC 媒体)、`ai`/`an`(启动坞)。`$expr$` 包裹求值；`ResolveText` 文本类同样依赖 `$`。

### 3.2 函数选择器弹窗（OpenFunctionPicker，xaml.cs:1097）
- 独立无边框透明置顶 `Window`，左右分栏：左侧分类 `catList`（`FunctionCatalog.All` 按 `Category` 去重），右侧函数 `fnPanel`（显示 `Sig`+`Desc`，点击插入 `fn.Insert` 到公式框光标处）。
- **现状**：基础分类列表已有，但呈现为简单列表，**非「网格卡片」**，且未与实时预览联动。

### 3.3 FunctionCatalog 分类（10 类，与 FunctionRegistry ~19 函数一一对应）
`func.cat.time`(df/tf/ts/tu/tz) · `func.cat.system`(si/bi/dp) · `func.cat.var`(gv) · `func.cat.logic`(if) · `func.cat.text`(tc/uc/re) · `func.cat.media`(mi) · `func.cat.math`(mu) · `func.cat.color`(ce/bp) · `func.cat.data`(wg) · `func.cat.misc`(fl/rng)。

**与设计差距**：缺「实时预览区」，函数网格呈现待升级为卡片式。

## 6. 流程 / 行为系统（Actions/）

| 项 | 现状 |
|----|------|
| Flow 结构 | `Flow.cs:22`：`Condition`(string 布尔公式) + `List<AtomAction> Actions` + `FlowFireMode Mode`(Once/While)。序列化用控制字符 `\u0001`(字段)`\u0002`(流程)`\u0003`(动作)（`Serialize/Parse` `Flow.cs:54-105`） |
| AtomAction 枚举 | `AtomAction.cs:9-25`：`None/RunApp/MediaControl/SwitchPage/ToggleEditMode/OpenSettings/LockScreen/OpenURL/Command/SwitchPreset` **(旧9种)** + `RunFlow/SetVar/Delay/ReadFile` **(新增4种，全已存在)** |
| ActionRunner | `ActionRunner.cs:19`：`RunAsync/RunAllAsync`(async，Delay 需 await)。`RunFlow` 经 `host.Flows[idx]` 递归 + `HashSet<int> visiting` 防循环(`ActionRunner.cs:55-67`)；`SetVar` 写 `win.Gv`(`:128`)；`ReadFile` 读 txt/json(支持 `a.b.c`/`a[0].b` 提取)写 gv(`:152`) |
| 动画系统 | `Atoms/Atom.cs`：`PlayAnimations`(:519)、`StartAnimationSequence`(进场 Fade/Slide/Zoom/Drop)、`StartLoopAnimation`(循环 Pulse/Rotate/Blink/Float/Bounce)、`BuildForeverLoop`(:676)。触发 `animWhen`(布尔公式)上升沿起播、为假复位(`EvalAnimWhen` :548、`TickAnimation` :539 由 `DirtyScheduler` 每拍调) |
| 触发器评估 | `Flow.ShouldFire(EvalContext)`(`Flow.cs:35`)：Once=上升沿，While=持续每周期。`Atom.EvaluateFlows()`(`Atom.cs:454`，仅桌面模式、编辑态不触发)由 `DirtyScheduler.Tick` 遍历全部原子 |

**已知权衡**：`RunFlow` 按**序号**引用本原子流程，流程增删/重排会序号漂移（已接受，无自动重映射）。组件变量 `cg()` 写侧仅属性面板手填，函数公式与变量体系待重构（用户拍板）。

## 9. 已知技术债 / 待重构项

1. **Tab 分类法不一致**：当前 `content/style/layout/...` 与 KLWP「形状/油漆/图层/位置/动画/触摸」语义不对齐，重构需重映射 + 新增「图层」页（承载混合/滤镜/模糊）。
2. **控件风格不统一**：裸 TextBox 为主，无步进器；Slider 联动 TextBox 是唯一定量控件。
3. **公式绑定视觉缺失**：无「已绑公式」方块提示，用户难一眼识别动态字段。
4. **窗口分散**：TreeWindow/PropWindow 双浮窗 vs KLWP 三栏一体；内嵌化需重构窗口结构。
5. **组件变量体系 preliminary**：`cg()` 解析链已落地但写侧仅手填，函数公式与变量体系待用户拍板重构。
6. **RunFlow 序号引用漂移**：增删/重排流程会令序号错位（已接受）。
7. **图层能力缺失**：混合/滤镜/模糊（v1.x 增强候选），且当前「质感」仅 Shape 的 14 程序化纹理，非图层级。
