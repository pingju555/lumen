# 渲染架构设计（Desktop Overlay Layer · WPF 实现深潜）

> 配套 `功能需求.md` §3（画布/网格）、§5（原子化框架）。将需求映射为 WPF 具体渲染 / 命中模型。
> 本栈已切 WPF（见 `技术栈选型.md` v2）。本文是"HOW"，`功能需求.md` 是"WHAT"。

---

## 0. 渲染管线总览

- WPF 为 **retained-mode**：代码 / XAML 构建视觉树 → Composition 线程按脏区重绘；无显式 Render Loop。
- 覆盖层渲染链路：

```
Config(JSON) → Model(Layer / Widget / Atom) → BuildVisual() → WPF Visual Tree → DWM 合成到屏幕
```

- 公式 / 动画驱动 `Invalidate`（改 DependencyProperty）→ WPF 自动重绘脏区；空闲无变更 → 零重绘（与 NFR-02 / NFR-07 天然契合，无需手动降帧也能 idle-cheap）。
- 易错点：不要自己开 `CompositionTarget.Rendering` 循环做全屏重绘；只改 DP，让 WPF 决定重绘范围。

## 1. 原子渲染抽象（§5）

### 1.1 Atom 基类

```csharp
abstract class Atom {
    string Id;
    Rect   Bounds;                       // (x,y,w,h) 局部 px，相对父容器原点，f32 亚像素
    int    ZIndex;
    double Opacity;
    BlendMode Blend;                      // Normal / Multiply / Screen / Overlay
    InputMode Input;                      // Block / PassThrough / InteractiveOnly
    Visibility Vis;
    Dictionary<string, IAtomProperty> Props;

    FrameworkElement BuildVisual(FormulaContext ctx);   // 构造/返回 WPF 元素
    void Evaluate(FormulaContext ctx);                   // 解析三元组 → 写回元素属性
}
```

### 1.2 属性三元组

```csharp
enum PropSource { Static, GlobalVar, Formula }

interface IAtomProperty { object Resolve(FormulaContext ctx); }

class AtomProperty<T> : IAtomProperty {
    PropSource Source;
    T        StaticValue;
    string   GvRef;      // 全局变量键
    string   Formula;    // $...$ 公式串
    T Resolve(FormulaContext ctx) {
        return Source switch {
            Static    => StaticValue,
            GlobalVar => ctx.Globals.Get<T>(GvRef),
            Formula   => ctx.Eval<T>(Formula),
            _         => StaticValue
        };
    }
}
```

- 解析时机：配置加载（静态值）、`gv` 变更、公式 tick（定时 / 事件触发）。
- 错误回退：`Resolve` 抛错 → 回退 `StaticValue` 或类型 default，不崩（FR-ATOM-15）。
- 持久化：`{type, x, y, w, h, z, props:{name:{source,value/gv/formula}}}`（见 `功能需求.md` §3.0）。

### 1.3 六类原子 → WPF 映射

| 原子 | WPF 元素 | 关键属性绑定 |
|------|----------|--------------|
| `Text` | `TextBlock`（弧形用 `FormattedText` 沿 `PathGeometry` 或 `Glyphs`） | Text / FontSize / Foreground / FontFamily / TextAlignment |
| `Shape` | `Rectangle` / `Ellipse` / `Line`，`Fill`=`SolidColorBrush`/`LinearGradientBrush`/`ImageBrush` | Geometry / Fill / Stroke / RadiusX/Y |
| `Icon` | `TextBlock`（字体图标：Segoe MDL2 / FontAwesome）/ 或 `Path`（矢量字形） | Text(Glyph) / FontSize / Foreground |
| `Image` | `Image`(`BitmapImage`)，圆角用 `Border` 包 + `CornerRadius` | Source / Stretch / 裁剪 |
| `Progress` | `ProgressBar` 或 `Shape`+`Clip`（环形用 `Arc` Path） | Value / Maximum / Orientation / Foreground |
| `Container` | `Panel`：`Overlap`→`Canvas`、`Stack`→`StackPanel`、`Sequence`→`UniformGrid`/方向 `StackPanel` | 子原子树、Padding、ZIndex |

- `Container` 渲染递归：自身 `BuildVisual` 遍历子 `Atom.BuildVisual` 加入自身 `Panel`。
- 渐变 / 开关 / 表单不是独立原子：渐变 = `Shape.Fill` 属性，开关/表单 = §1.7 行为系统（与 `功能需求.md` §6.0 一致）。

### 1.4 视觉树装配

```
Window
 └ RootPanel (Grid，铺满 WorkArea)
    └ 每个 Layer → ContentPresenter / Panel
       ├ CanvasMode : <Canvas>          原子绝对定位 (Canvas.SetLeft/Top)
       └ GridMode  : <UniformGrid>      cell → 每 cell 一个 WidgetHost
            └ WidgetHost → 内嵌 <Canvas> 放原子树（局部 px）
```

- z-order：设 `Panel.ZIndex`（Layer 间 + 同 Layer 原子间，FR-CANVAS-03 / FR-LAYER-01）。
- 混合：Normal → `Opacity`；Multiply/Screen → `UIElement.Effect = new BlendShaderEffect(mode)`（见 `技术栈选型.md` §6.1 ShaderEffect）。
- 同屏要画布 + 网格并存 → 建两个层（画布层 + 网格层）按 z 合成（FR 已定）。

### 1.5 脏标记 / 重绘

- WPF 自动：改 DependencyProperty → 自动重绘脏区（retained-mode）。
- 公式 tick：定时器 / 事件 → `atom.Evaluate(ctx)` → 写回 DP → WPF 重绘。
- 无需手写脏标记框架；保留"脏触发"概念仅用于决定何时跑 `Evaluate`（见 §4 降频）。

## 2. 画布 / 网格系统（§3）

### 2.1 坐标系（三层嵌套）

| 坐标 | 原点 | 单位 |
|------|------|------|
| Screen | 工作区左上 | px |
| Canvas（Layer-local） | Layer 原点 | px（f32，亚像素） |
| Grid-cell | 网格原点 | cell |
| Widget-local | cell 盒左上 | px（f32） |

- Canvas 原子：`(x,y,w,h)` 相对 Layer 原点。
- Grid 单元：`cellRect(r,c) = origin + (c*spacing, r*spacing, spacing, spacing)`。
- Widget 局部：相对其 cell 盒左上。

### 2.2 Canvas 模式（Px 画布，FR-CANVAS-*）

- 容器：WPF `<Canvas>`。原子用 `Canvas.SetLeft/Top(Bounds.X/Y)` + `Width/Height`。
- 自由拖拽 / resize：编辑态 adorner 手柄（§2.4）；运行态无手柄。
- 无强制吸附（可开对齐辅助线，FR-CANVAS-02）。

### 2.3 Grid 模式（容纳小组件，FR-GRID-*）

- 容器：`<UniformGrid Rows=.. Columns=..>`；`cellSize = spacing`（由分辨率 gcd 推导，四档 {20,40,60,120}，FR-GRID-10）。
- 小组件锚定：`Grid.SetRow/Column/RowSpan/ColumnSpan` → 映射到 UniformGrid 单元。
- Widget 内部：每 cell 放 `WidgetHost`(`Border`/`ContentPresenter`) → 内嵌 `<Canvas>` 放原子树（局部 px）。
- 拖拽吸附：落点 `row = floor((y-originY)/spacing)`，`col = floor((x-originX)/spacing)`；跨行列 resize 改 RowSpan/ColumnSpan。
- 单元互斥：放置前检测 span 重叠，禁止（FR-GRID-06）。

### 2.4 编辑态外壳 / 手柄

- `AdornerLayer`：选中原子 / Widget 显示移动 + 缩放手柄 + 标题栏（FR-ATOM-16）。
- 对齐参考线：编辑态叠加 `Line` 标尺（FR-CANVAS-04 / FR-GRID-03）。
- 运行态：清空 `AdornerLayer`，视觉树仅留内容。

### 2.5 响应式 / 多屏

- Canvas px 固定；换屏策略（固定 px 或百分比锚定）待定。
- Grid cell 按屏占比：spacing 候选集随分辨率 gcd 自适应（FR-GRID-08/10）。
- 多实例各屏独立层栈（多开模型，FR-COV-06/07）。

## 3. 输入与点击穿透（覆盖层核心难点）

### 3.1 窗口级穿透

- 纯 `WS_EX_TRANSPARENT` 会让整窗鼠标事件穿透到下层（桌面 / 应用）。
- 但 FR-COV-04 要求空白区"拦截输入、不向桌面穿透"——与纯穿透矛盾。两种方案：

| 方案 | 做法 | 取舍 |
|------|------|------|
| **A（v1 推荐）** | 窗口**不**全局穿透；普通原子 `IsHitTestVisible=false`（不接收但也"挡住"桌面点击，即空白被吸收），交互原子 `IsHitTestVisible=true` 接收点击 | 纯 WPF 即可，无 WndProc hack；空白点击被窗口吞掉（满足 FR-COV-04 拦截语义），按钮可点 |
| **B** | 窗口全局 `WS_EX_TRANSPARENT`；交互原子用命中测试"打洞"（WndProc 在交互矩形内返回 `HTCLIENT`，否则 `HTTRANSPARENT`） | 真正透明窗口，空白点击可透到桌面；实现复杂（需自管 WndProc + 命中矩形集） |

- **取舍结论**：v1 用方案 A。交互原子显式 `IsHitTestVisible=true`，其余 `false`。空白点击被窗口吸收（不传到桌面），满足 FR-COV-04/05；桌面图标被盖且点不到。
- 若后续用户期望"点空白能进桌面"，再切方案 B（对应 FR 语义需同步改为 PassThrough）。

### 3.2 非置顶层级（FR-COV-02）

- 覆盖窗口 `Topmost=false`；通过 `SetParent` 到桌面（`Progman` / `SHELLDLL_DefView`）或特殊 z-order，使其位于桌面图标之上、普通应用之下。
- 窗口铺 `SystemParameters.WorkArea`（排除任务栏），任务栏保持可交互（FR-COV-08/09）。
- 显隐（FR-COV-11）：全局快捷键 → 隐藏=收起窗口并释放输入，显示=恢复。

### 3.3 输入合成（多层，FR-LAYER-04）

- 每层 `InputMode`：Block（整层消费）/ PassThrough（整层穿透）/ InteractiveOnly（仅交互原子接收）。
- 命中测试自顶向下：首个 `IsHitTestVisible=true` 的交互元素消费；否则按层 InputMode 决定穿透或拦截。

## 4. 与 NFR-07 降频衔接

- `DispatcherTimer` 驱动公式 tick（时钟 1s、监控按 NFR 刷新率、媒体订阅 SMTC 事件）。
- 获焦 + 有动态内容：满帧 tick。
- 失焦：Interval 降到 ~250ms（≈4fps），维持时钟 / 监控更新但不浪费。
- 失焦且无动态（无动画、无活跃公式 tick）：`Stop()` timer，纯事件驱动刷新。
- 重新获焦：`Start()` timer 恢复。
- WPF retained-mode 保证无 DP 变更时 Composition 线程不重绘 → 空闲 CPU 近 0（NFR-02/07 协同）。

## 5. 待确认 / 风险

- **毛玻璃**：需 `DwmEnableBlurBehindWindow` P/Invoke；WPF `Window` 须 layered 且不画背景（与 §2 背景层 FR-BG-03 对接）。
- **混合模式**：`BlendShaderEffect` 需写 HLSL（`PixelShader`）；v1 仅 Normal + Opacity 可先不做真实混合。
- **点击穿透语义**：方案 A 下空白点击被吞（不进桌面）。若 FR-COV-04 改为"允许穿透桌面"，需切方案 B。
- **弧形文字（CurvedText）**：`Text` 子模式，WPF 无原生，需 `FormattedText` + `PathGeometry` 或 `Glyphs` 自绘，v1 可降级为直线。
- **Container 嵌套性能**：深层原子树 + 频繁公式 tick 时，注意只 `Invalidate` 受影响子树，勿整体重 build 视觉树。
