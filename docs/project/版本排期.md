# 版本排期（Version Roadmap）

> 配套 `v1开发计划.md`（Phase 级实现拆解 P0–P6）、`技术栈选型.md`、`功能需求.md`。
> 本文档将 P0–P6 映射到**发布版本**（v1.0 / v1.1 / v1.2 / v2.0），定义每个版本的主题、范围、交付与出口标准。
> 版本策略：**每个版本可独立发布、可运行、向后兼容配置 schema（版本号迁移，见 `技术栈选型.md` §5.1）**。

## 0. 版本策略
- 语义化：`v1.0` 首个可用版；`v1.x` 增量（不破坏配置）；`v2.0` 含 deferred 大特性 / 架构跃迁。
- 每版本 = 一组 Phase 的聚合；Phase 细节见 `v1开发计划.md`。
- 配置 schema 带版本号，升级向前兼容（手写迁移，`技术栈选型.md` §5.1）。
- 体积 / NFR 门禁：每个版本 Release 抽检（<80MB 或 `技术栈选型.md` §4 取舍）。

## 2. v1.0 — 核心覆盖层（可用 MVP）
**目标**：一个能取代原生桌面的原子化覆盖层——放六类原子、实时公式时钟、配置可存。
**范围（Phase）**：P0 + P1 + P2，加为"可用"必须的最小背景/层/设置/持久化。
**交付**
- 覆盖窗口：非置顶、任务栏系统 Auto-hide 铺 WorkArea 不预留、输入拦截（方案 B `WS_EX_TRANSPARENT`+命中测试备选，`技术栈选型.md` §3.1/§6）、v1 锁定单屏 1920×1080（FR-COV-01~05, 08~12）。
- 守护：双模式 exe + 互相看门（管道心跳 + clean-shutdown 事件）+ 注册表自启（FR-COV-15/17，§3）。
- 多层：三类预设（壁纸层 / 网格化层 / 原子化 Px 画布层）骨架 + z / opacity / enabled（FR-LAYER-01~03）。
- Px 画布：原子 `(x,y,w,h)` 自由定位 + 标尺/参考线（FR-CANVAS-01~05）。
- 六类原子 + 属性三元组（静态 / gv / 公式）+ 公式引擎 19 函数 + gv（FR-ATOM-*、公式引擎设计）。
- 持久化：每屏 × 每层 JSON（FR-COV-06/07、§3.0、FR-SET-08）。
- 背景最小集：图片(填充/适应/平铺/居中/拉伸) / 纯色 / 线性·径向渐变（FR-BG-01/02/04）。
- 托盘（`Hardcodet.NotifyIcon.Wpf`）+ 显隐快捷键 + 渲染降频（FR-COV-11、NFR-07）。
- 设置面板最小集：显隐快捷键、背景、自启（FR-SET-01/02/04/05）。
**出口标准**：双击 exe → 全屏覆盖层出现，图标/壁纸被盖且不可交互；放一个 Text 显示 `$df(hh:mm:ss)$` 实时跑；配置存盘重开一致；杀 UI 秒级自拉起；常驻内存达标（NFR-01 抽检）。
**不含**：网格 / 小组件 / 页面、实时数据(si/mi/mu/ai/an)、行为系统、毛玻璃、导入导出。
**依赖**：无前置，起点。

## 3. v1.1 — 网格 / 小组件 / 页面
**目标**：从"放原子"升级到"组织部件"，多页面工作区。
**范围**：P3。
**交付**
- 网格：gcd 间距四档 {20,40,60,120}、cell→px、吸附、跨行列 resize、冲突禁止（FR-GRID-01~12）。
- 小组件：原子树打包入库（`WIDGET_REGISTRY`）、拖入 cell、整体移动/resize/显隐、嵌套、属性继承（FR-WIDGET-01~07）。
- 页面：多页、独立网格+背景、切换（指示/滚轮/快捷键）+ 过渡、管理(增删改名排序)、上限默认 9（FR-PAGE-01~06）。
- 预设库：内置固定预设（三类层）+ 用户自定义预设 增删改导入导出（FR-LAYER-08~10）。
**出口标准**：打包「时钟小组件」拖入网格；多页切换无闪烁；自定义预设可复用/导出。
**依赖**：v1.0（Px 画布 + 原子 + 持久化前置）。
**配置**：schema v1 → v1.1 迁移（加 grid / widget / page 字段）。

## 4. v1.2 — 实时数据 / 行为 / 打磨
**目标**：覆盖层真正"活"起来——真实系统数据 + 交互行为 + 体积性能达标。
**范围**：P4 + P5 + P6 余下。
**交付**
- 数据接入：`si`(PerformanceCounter/PDH) / `mi·mu`(SMTC) / `ai·an`(开始菜单 `.lnk` + UWP `PackageManager`)（§6 系统监控/媒体/启动坞，FR-SYS-*/FR-MEDIA-*/FR-LAUNCH-*）。
- 行为系统：动画(进入/退出/循环·缓动/延迟) + 触发器(显示/点击/悬停/空闲/定时/系统事件) + 按钮行为(切层/切页/运行/快捷键/动画/URL/脚本)（FR-BEH-01~10）。
- 背景：毛玻璃 `DwmEnableBlurBehindWindow` + 每页独立背景 + 切换过渡（FR-BG-03/05）。
- 设置闭环：网格间距 / 背景 / 多屏 / 配置导入导出（FR-SET-03/06/07）。
- 便签 / 待办 / 文件速览：XAML + Shell 图标/缩略图(`SHGetFileInfo`/`IShellItemImageFactory`) + `FileSystemWatcher`（FR-NOTE/TODO/FILE-*）。
- 流程工作流：触发器 → 步骤(延迟/条件) → 动作，命令行 TTS/音频（FR-WF-01~08）。
- 打磨：体积核对、空闲 CPU~0、帧 <16ms、崩溃/被杀自恢复（NFR-01~07）。
**出口标准**：系统监控/媒体/启动坞实时出数据；点图标启动程序；空闲触发显隐；毛玻璃生效；配置导入导出无丢失；Release exe 体积达标；常驻内存达标。
**依赖**：v1.1（组件/页面作为行为载体）。
**配置**：schema v1.1 → v1.2（加 behavior / widget-data / file / workflow 字段）。

## 5. v2.0 — 扩展与多屏（deferred 分支）
**目标**：释放 v1 锁定的 deferred 路径 + 架构扩展。
**范围**：`功能需求.md` §6.5/§6.6 / FR-CW 天气 / FR-BG-06 / FR-COV-06 多实例 / FR-ATOM-11 插件化。
**交付（按优先级，可拆 v2.x）**
- 多屏多实例：单进程多 `Window` 按屏覆盖，每屏独立层栈（FR-COV-06/07，技术路径 `技术栈选型.md` §5.2）。
- 天气：外部 API（待数据源/Key 拍板）→ `HttpClient`（FR-CW-06~09）。
- 网页部件：WPF `WebView2`（`Microsoft.UI.WebView2`，不切 Tauri/Electron，§5.2）。
- 本地 LLM：原生 `HttpClient` + XAML 对话 UI（§6.6）。
- 动态壁纸/视频：FR-BG-06（v1 不做，v2 评估）。
- 原子插件化动态加载：FR-ATOM-11（编译期 → 插件式）。
- 混合模式精化 + PDH 精度（`v1开发计划.md` 备注遗留）。
**依赖**：v1.2 全量稳定。
**风险/开放**：天气数据源与 Key、WebView2 运行时分发、LLM 本地服务形态待用户拍板（`功能需求.md` §9 待决）。

## 6. 跨版本横切
- **NFR 门禁**：每版本 Release 必过 体积(<80MB 或 §4 取舍) / 空闲 CPU~0 / 帧 <16ms / 崩溃自恢复。
- **测试**：v1.0 起公式引擎 xUnit 单测；v1.2 补行为/数据接入单测；UI 以手工验收为主。
- **配置兼容**：schema 版本号迁移，破坏性变更仅在 v2.0。
- **打包**：`dotnet publish`（框架依赖 A / 自包含裁剪 B 取舍，`技术栈选型.md` §4）。

## 7. 依赖与顺序（文本甘特）
```
v1.0 ──► v1.1 ──► v1.2 ──► v2.0
(P0-2)   (P3)      (P4-6)    (deferred)
  │         │          │
  └─ 每个版本内含 守护 / 持久化 / 降频 抽检
```
- v1.0 不依赖其他版本，是后续基座。
- v1.1 依赖 v1.0 的画布/原子/持久化。
- v1.2 依赖 v1.1 的组件/页面作为行为载体。
- v2.0 依赖 v1.2 稳定，且多项需用户拍板外部依赖。

## 8. 开放决策与风险
- v1.0 是否含"毛玻璃 / 导入导出"？当前排期放 v1.2；若想 v1.0 即更完整可前移（代价：v1.0 周期拉长）。
- 多屏：v1 锁单屏，v2 多实例——确认"单进程多 Window"路径（§5.2）是否接受，或改多进程。
- deferred 三项（天气/网页/LLM）需外部依赖拍板，可能在 v2.0 进一步拆 v2.1 / v2.2。
- NFR-01 体积取舍（§4）影响发布形态，建议 v1.0 早期定。
