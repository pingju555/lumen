# 桌面覆盖层 · 功能需求详述

> 配套 `项目规格.md`。将 §5 候选清单展开为可执行的功能需求（FR 编号便于追溯）。

## 1. 覆盖层（Cover Layer）
### 1.1 遮挡与输入
- **FR-COV-01** 覆盖窗口铺满目标显示器工作区（任务栏自动隐藏时不预留，铺满全视觉区域），覆盖原生桌面图标与壁纸；**v1 锁定单屏 1920×1080**（多屏配置结构见 FR-COV-06，v1 不强制多实例）。
- **FR-COV-02** 窗口层级位于原生桌面图标层之上、普通应用窗口之下（**非全局置顶**）：覆盖桌面图标与壁纸，但不抢占普通应用的 z-order；任务栏由系统自动隐藏，其弹出时浮于覆盖层之上（覆盖层局部被遮挡，可接受）。
- **FR-COV-03** 无边框、无标题栏、无系统菜单。
- **FR-COV-04** 拦截鼠标/键盘输入，不向原生桌面穿透。
- **FR-COV-05** 原生桌面图标与壁纸被背景层完全遮盖，不可见。

### 1.2 多显示器
- **FR-COV-06** 多显示器配置**按屏幕持久化**：配置结构为「每屏 → 每层」（`{ screens:[{ id, layers:[...] }] }`），各屏层栈独立；v1 锁定单屏 1920×1080，但 schema 已支持多屏，后续可多开实例覆盖各屏。
- **FR-COV-07** 当前单屏 → 单实例覆盖主屏；每屏层栈独立（见 FR-COV-06）。各屏背景/网格可独立或统一（配置项）。

### 1.3 任务栏（系统自动隐藏 + 弹出浮于覆盖层之上）
**决策（用户确认，2026-07-15 修订）**：任务栏**由系统自动隐藏（Auto-hide）**，覆盖层铺满工作区、不预留任务栏区域；任务栏弹出时浮于覆盖层之上，覆盖层局部被遮挡可接受。覆盖层**不接管**任务栏显隐（系统自管）。
- **FR-COV-08** 依赖系统任务栏自动隐藏：覆盖层铺 `SystemParameters.WorkArea`（自动隐藏场景下即全视觉区域），**不单独探测/预留任务栏尺寸**；启动时探测任务栏是否处于自动隐藏态以正确铺满。
- **FR-COV-09** 任务栏由系统自动隐藏管理：鼠标贴近屏幕下边缘时系统唤起任务栏，浮于覆盖层之上；覆盖层非全局置顶（`Topmost=False`），保证任务栏弹出时层级在其上。
- **FR-COV-10** 任务栏在左/右/上侧自动隐藏时同理按系统行为处理；覆盖层不针对某边做特殊预留。
- 显隐快捷键**保留**（用户澄清 2026-07-15），见 §1.4 FR-COV-11。

### 1.4 显隐切换
- **FR-COV-11** 全局快捷键隐藏/显示覆盖层（回到原生桌面）。
- **FR-COV-12** 覆盖层隐藏时完全释放输入拦截，原生桌面恢复全部交互。
- **FR-COV-13** 热角唤起（鼠标推至屏幕角）可选，默认关。
- **FR-COV-14** 显隐过程有淡入/淡出过渡（< 200ms）。

### 1.5 生命周期
- **FR-COV-15** 开机自启可选（写入注册表 `Run`）。
- **FR-COV-16** 常驻后台，空闲时近零资源占用。
- **FR-COV-17** 崩溃后自动重启（可选，守护进程）。

## 1.6 多层系统（Layers）
> 覆盖窗口由多层堆叠合成（见 `项目规格.md` §4.1）。每层独立可配置，支持进阶操作。
- **FR-LAYER-01** 层可增删、排序（z-order 调整）。
- **FR-LAYER-02** 每层通用属性可配：启用/禁用、不透明度、混合模式、输入模式、绑定范围、显隐规则、视觉样式、更新策略（对应规格 §4.1 配置表）。
- **FR-LAYER-03** 层类型（预设三类 + 可扩展）：**壁纸层**（仅一个背景：图片 / 纯色 / 渐变 / 模糊，位于最底层）/ **网格化层**（网格格点盛放网格部件）/ **原子化 Px 画布层**（Px 画布自由放原子）；效果层 / 交互层等后续可扩展。
- **FR-LAYER-04** 输入按各层输入模式合成：拦截 / 穿透 / 仅部件区域可交互；穿透层不消费输入。
- **FR-LAYER-05** 层显隐规则：常显 / 按页面 / 按快捷键 / 自动（如空闲触发）。
- **FR-LAYER-06** 层配置持久化（与布局一同存本地文件），**含动画/触发器/按钮行为配置**（见 §1.7）。
- **FR-LAYER-07** 层参数在设置面板调参即时预览（实时生效）。
- **FR-LAYER-08** 层添加采用**预设**快速添加，预设分两类：(a) 内置固定预设（壁纸层 / 网格化层 / 原子化 Px 画布层），预填 9 项通用属性与默认行为；(b) 用户自定义预设（见 FR-LAYER-09）。添加后可微调。
- **FR-LAYER-09** 用户可将当前层配置（含 9 项通用属性与行为）保存为**自定义预设**存入预设库；后续添加层时可从库选择复用；自定义预设随配置持久化（见 FR-LAYER-06）。
- **FR-LAYER-10** 预设库管理：重命名 / 删除 / 导出 / 导入自定义预设。

## 3. 布局系统：Px 画布 + 网格（Canvas + Grid）
> **架构变更（用户补充）**：原子化后，原子不再绑定网格，改用 **基于 Px 的细致画布** 自由定位（像素级精细控制）。**网格系统保留，但其内容是「小组件」**（原子的可复用组合，见 §5.7），而非原始原子。两层关系：Px 画布放原子 → 原子打包成小组件 → 小组件进网格单元。

### 3.0 布局模型详解

布局由 **三层嵌套** 构成：窗口 → 层 → 放置面（画布 / 网格）。原子与小组件分属两个放置面，**互不混放**。

**坐标系**
- **Screen**：覆盖窗口客户区（任务栏系统自动隐藏，铺满工作区不预留），原点左上，单位 px。
- **Canvas**：原子自由定位空间，单位 px（`f32`，支持亚像素）；原子以 `(x,y,w,h)` 定位，相对层原点。
- **Grid**：逻辑单元空间，单位 cell；小组件以 `(row,col,rowSpan,colSpan)` 定位；单元 → px 矩形由「网格原点 + 网格间距（单元边长，见 §3.2.1）」算出；行数 = `H/spacing`、列数 = `W/spacing`。

**容器层级**
```
Window
 └ Layer (z / opacity / blend / inputMode / 9 配置项)
    ├─ CanvasMode : Atom* (px)        [Container 子原子亦为局部 px]
    └─ GridMode   : Widget* (cell) → Widget 内部 = Atom 树（自有 px 子画布）
```
- 一个内容层取 Canvas 或 Grid 其一；同屏要两者并存 → 建两个层（画布层 + 网格层），按 z-order 合成。
- Widget 内部最终仍由原子渲染（px 子画布）；网格只决定 Widget 在屏上的「盒子」位置，不干预内部原子布局。

**原子定位（画布）**
- 绝对 px `(x,y,w,h)`，支持小数；可选 anchor 与对齐辅助线（编辑态）。
- 同层原子 z 序可调（FR-CANVAS-03）。
- Container 原子自身为 px 盒，子原子坐标相对其盒（局部坐标），形成嵌套树。

**小组件定位（网格）**
- 逻辑 cell `(row,col)` + span → 映射 px 盒（合并 cell 矩形）。
- Widget 内部原子按该盒的**局部 px** 坐标布局；整体移动 / resize = 改 cell 或 span，内部原子坐标随之平移。
- 同 cell 互斥，span 重叠禁止（FR-GRID-06）。

**编辑模型**
- 画布态：拖原子（px 自由）、手柄 resize、标尺 / 参考线（FR-CANVAS-04）。
- 网格态：拖 widget（吸附 cell）、跨行列 resize（FR-GRID-04/05）。
- 小组件设计：在画布摆内部原子 → 打包 → 存预设 → 拖入网格。
- 空白区：画布与网格空白均消费输入（不穿透原生桌面）。

**持久化影响**
- Atom：`{type, x, y, w, h, z, props(三元组)}`
- Widget：`{id, row, col, rowSpan, colSpan, atomTree, inheritedProps}`
- Layer：`{contentMode: canvas|grid, ...9 项}`
- Grid：`{rows, cols, cellSize|screenRatio, origin}`
- Screen：`{ id, layers:[ Layer ] }`（每屏独立层栈，见 FR-COV-06）
- 整体配置根：`{ screens:[ Screen ], globals, ... }`（每屏 × 每层持久化）

**响应式 / 多屏**
- Canvas px 固定；换屏 / 分辨率可选「固定 px」或「百分比锚定」重定位（策略待定）。
- Grid cell 可按屏占比自适应。
- 多实例各屏独立层栈（多开模型，FR-COV-06/07）。

### 3.1 Px 画布（原子放置面）
- **FR-CANVAS-01** 覆盖窗口坐标空间 = 像素（px）；原子以 `(x, y, w, h)` 像素自由定位，支持小数 px。
- **FR-CANVAS-02** 自由拖拽 / 手柄 resize，无强制吸附（可开可选对齐辅助线）。
- **FR-CANVAS-03** 同层内原子层级（z-order）可调，决定叠放先后。
- **FR-CANVAS-04** 编辑态显示 px 标尺 / 对齐参考线；运行态隐藏。
- **FR-CANVAS-05** 空白区域消费输入（不穿透原生桌面）；右键开画布菜单（加原子 / 加小组件 / 切编辑态）。

### 3.2 网格系统（容纳小组件）
- **FR-GRID-01** 网格单元尺寸 = 网格间距（px）；行数 = `H/spacing`、列数 = `W/spacing`（由间距推导，非独立配置，保证整除无余数）。间距候选集见 §3.2.1。
- **FR-GRID-02** **小组件**锚定到 `(row,col)`，支持 `rowSpan`/`colSpan` 跨单元（原子不进网格，只进画布）。
- **FR-GRID-03** 编辑态显示网格线与单元高亮；运行态隐藏。
- **FR-GRID-04** 拖拽小组件到目标单元，自动吸附对齐。
- **FR-GRID-05** 拖拽边缘手柄调整跨行/跨列（resize）。
- **FR-GRID-06** 单元冲突：禁止重叠，占用单元显示占位提示。
- **FR-GRID-07** 空白单元消费输入（不穿透到原生桌面）；右键/长按打开网格菜单（添加小组件、切换编辑态）。

### 3.2.1 网格间距（Spacing）
> 网格间距 = 相邻格点 px 距离 = 单元边长。用户约束（以 1920×1080 为例）：取分辨率两值的**公约数**（保证整除对齐），最小 20、最大不超过 120、且为 **10 的整倍数**。

- **FR-GRID-08** 间距候选集 = `{ d : d | gcd(W,H), d % 10 == 0, 20 ≤ d ≤ 120 }`，由分辨率自适应生成。
- **FR-GRID-09** 以 1920×1080 为例：`gcd(1920,1080)=120` → 候选 `{20, 30, 40, 60, 120}`（均整除两维：96×54 / 64×36 / 48×27 / 32×18 / 16×9 格）。
- **FR-GRID-10** **已定稿**：间距四档 `{20, 40, 60, 120}`（30 合法但明确不纳入，保持档位清爽）；可下拉或快捷键循环切换。
- **FR-GRID-11** 间距变更即时重算 cell 矩阵，已放置小组件按 cell 重新吸附（坐标映射，不丢布局）。
- **FR-GRID-12** 默认间距可配（建议 120 或 60）。
- **边缘情况**：若 `gcd(W,H) < 20`（如 1366×768，gcd=2），候选集为空 → 回退策略待定（放宽到「整除较小边」或允许末格非满 / 固定取 20）。

## 5. 原子化部件框架（Atomic Widget Framework）

> 取代"整块部件 + 统一 Rust trait"旧模型。部件 = **最小可放置原子的递归树**（源自 KLWP 原子化思路，见 `KLWP 逆向分析.md`）。公式引擎 v1 即生效（见 `公式引擎设计.md`）。

### 5.1 原子类型（v1 六类打底）
- **FR-ATOM-01** v1 原子类型（对齐 Kustom 真实清单，见 `Kustom 原子模型.md` §2）= `Text`（文本，含 CurvedText 沿路径子模式）/ `Shape`（形状，填充含渐变/图片）/ `Icon`（字体图标）/ `Image`（位图）/ `Progress`（进度）/ `Container`（容器，三布局：重叠/堆叠/序列）。
- **FR-ATOM-02** 后续扩展 `Movie`（视频，对应 Kustom MovieModule）。`Komponent`（嵌套组合）作为**组合预设机制**（§5.3）而非基础原子。扩展点 = 原子注册表（§5.4）。
- **FR-ATOM-03** 每种原子定义自身可配置属性集（尺寸/位置/颜色/内容等）。

### 5.2 属性三元组（原子可变性的本质）
- **FR-ATOM-04** 每原子属性三选一绑定：**静态值 / 全局变量(`gv`) / 公式(`$...$`)**。
- **FR-ATOM-05** 公式分支自 v1 生效，由公式引擎求值（数据源全本地，见 `公式引擎设计.md`）。
- **FR-ATOM-06** 三元组随原子持久化进 JSON（§7）。

### 5.3 容器与组合
- **FR-ATOM-07** `Container` 可嵌套子原子（树），实现分组/堆叠，对应 KLWP 的 Overlap/Stack。
- **FR-ATOM-08** 网格部件层本身即布局容器；原子锚定到网格单元（§3）。
- **FR-ATOM-09** 小组件（komponent 思想，见 §5.7）：原子树可保存为可复用预设，从库拖入。

### 5.4 原子注册表
- **FR-ATOM-10** 原子声明：`{ typeId, name, defaultProps, schema, render }`。
- **FR-ATOM-11** v1 编译期注册；后续升级插件式动态加载（独立编译单元）。
- **FR-ATOM-12** 新增原子不破坏现有持久化与求值器（开放封闭）。

### 5.5 刷新与求值
- **FR-ATOM-13** 公式求值由数据变更/定时 tick 触发；脏标记增量重算。
- **FR-ATOM-14** 时钟 1s tick；系统指标由 `SystemProvider` 轮询 PDH/WMI 脏触发；音乐订阅 SMTC 事件。
- **FR-ATOM-15** 求值错误回退静态值/空，不崩溃。

### 5.6 编辑态外壳
- **FR-ATOM-16** 编辑态原子显示外壳：标题栏 + 移动/缩放手柄 + 配置钮 + 删除钮。
- **FR-ATOM-17** 运行态隐藏外壳，仅渲染内容。
- **FR-ATOM-18** 原子坐标基于 **Px 画布**（§3.1），以 `(x, y, w, h)` 像素定位；不绑定网格单元（网格只容纳小组件，见 §5.7）。

### 5.7 小组件（Widget）— 网格的内容单元
> **一等概念（用户引入）**：小组件 = 原子的可复用组合（原子树 + 默认三元组 + 布局），是网格系统（§3.2）的内容单元。等价于 KLWP 的 komponent 思想，但提升为一等概念，区别于底层原子。

- **FR-WIDGET-01** 小组件由若干原子组成（原子树），内部原子按 Px 画布定位。
- **FR-WIDGET-02** 小组件可保存为复用预设，进入小组件库，拖入即用（komponent 思想落地）。
- **FR-WIDGET-03** 小组件作为整体锚定到网格单元（§3.2），整体移动 / resize / 显隐。
- **FR-WIDGET-04** 小组件可嵌套（含子小组件），形成树形组合。
- **FR-WIDGET-05** 小组件属性（三元组）可被外层覆盖 / 继承（主题色、显隐规则、字体）。
- **FR-WIDGET-06** 内置小组件：时钟/日历、启动坞、系统监控、媒体、便签、文件速览（对应 §6 组合预设）。
- **FR-WIDGET-07** 小组件编辑：在 Px 画布上设计内部原子布局，打包后作为一个单元出入网格。

## 6. 具体部件功能需求（v1 原子集 + 组合预设）

### 6.0 v1 原子集（最小可放置单元）

| 原子 | 核心属性（三元组） | 典型用途 |
|------|-------------------|----------|
| `Text` | 内容(formula/static)、字号、颜色、对齐、字体、沿路径 | 时钟、日历文字、便签、标题、弧形文字 |
| `Shape` | 几何(矩形/圆/线)、尺寸、填充(纯色/渐变/图片)、圆角、描边 | 背景块、分隔线、进度底、卡片 |
| `Icon` | 字形(字体图标集)、尺寸、颜色 | 启动坞图标、系统状态图标、天气图标 |
| `Image` | 源(位图/路径)、缩放、圆角、裁剪 | 头像、壁纸缩略、图片便签 |
| `Progress` | 当前值(formula)、最大、方向、颜色、样式 | 电量/内存/CPU/网速条 |
| `Container` | 子原子树、布局(重叠/堆叠/序列)、内边距、z序 | 分组、启动坞网格、监控面板、多页轮播 |

> 原子清单经 KLWP+KWGT 代码级确证（见 `Kustom 原子模型.md`）。渐变/开关/表单不是独立原子：渐变=Shape 填充属性，开关/表单=行为系统。

**v1 公式子集**（见 `公式引擎设计.md` §4）：`df tf si bi gv if tc mi mu`，覆盖时钟/系统监控/媒体；天气(`wi/wf`)等仅占位。

### 6.0.1 旧"整块部件" → 小组件映射

> 此前候选的整块部件（W1–W10）现重构为**小组件**（§5.7）——原子的可复用组合（原子树），作为网格内容单元。原子层（Px 画布）不变，仅上层组织方式改为「小组件进网格」。

| 原部件 | 原子组合方案 |
|--------|--------------|
| W1 时钟/日历 | `Text`(df/tf) + `Shape`(卡片) + `Container`(月视图网格) |
| W3 应用启动坞 | `Icon`(字形) + `Container`(网格) + `tc`(名称截断) |
| W4 系统监控 | `Progress`(si CPU/内存/网速) + `Text`(数值) + `Shape`(条底) |
| W7 媒体控制 | `Text`(mi title/artist) + `Icon`(播放/暂停) + `mu`(控制) |
| W5 便签 / W6 待办 | `Text` + `Shape`(卡片) + `Container` |
| W10 文件速览 | `Icon` + `Text` + `Container` |
| W2 天气 / W8 网页 / W9 本地LLM | 后续扩展（`Image`/`WebView` 容器），v1 不纳入 |

## 8. 非功能需求
- **NFR-01** 内存占用目标**尽量小**（起步 < 80 MB，严格偏紧）；具体优化后期详谈。空闲 CPU 近 0%。
- **NFR-02** 覆盖层静止时 CPU 占用近零。
- **NFR-03** 遮挡 / 切换 / 拖拽帧耗时 < 16ms（无明显卡顿）。
- **NFR-04** 兼容 Windows 10 / 11。
- **NFR-05** 崩溃不得导致原生桌面丢失（显隐安全，FR-COV-12 兜底）。
- **NFR-06** 配置损坏可一键恢复默认。
- **NFR-07** 基于焦点的渲染降频（性能优化）：窗口失去焦点时渲染降频至低帧（如 4fps）；当无任何动画/公式 tick 活动时停止主动重绘，仅由事件驱动刷新；窗口重新获得焦点且存在动态内容时恢复满帧。与 NFR-02（空闲 CPU 近 0）协同。

## 10. 流程工作流（Workflow）
> 新增模块（用户引入）：编排带时序与条件的多步自动化，扩展 §1.7 行为系统。例：点击按钮 → 等待 60 分钟 → 命令行触发语音"该喝水了"提醒。

### 10.1 工作流模型
- **FR-WF-01** 工作流由「触发器 → 步骤序列 → 动作」组成；步骤可含延迟（如 60min）与条件（时间窗/变量判断）。
- **FR-WF-02** 触发器复用 FR-BEH-05 事件源（显示层/隐藏层/点击/悬停/空闲 N 秒/定时 cron/系统事件/页面切换）+ 工作流间触发。
- **FR-WF-03** 步骤：每个步骤 = 一个动作 + 可选延迟(ms/min/h) + 可选条件；序列按序执行，并行分支待议。
- **FR-WF-04** 动作类型继承 FR-BEH-07（切换层/切页/运行程序/发快捷键/触发动画/打开 URL/调用本地 LLM/执行脚本）+ 重点「运行命令行」。
- **FR-WF-05** 「运行命令行」动作：执行指定命令行（可配置同步/异步、工作目录、参数）；用于触发系统能力（如语音播放）。
- **FR-WF-06** 语音提醒示例：命令行调用系统 TTS（如 PowerShell `Speech.Synthesis` 朗读文本）或播放预录音频文件（如 `Media.SoundPlayer`），两者皆支持；文本/文件路径可配置。
- **FR-WF-07** 工作流持久化（随配置存 JSON）、管理（增删 / 启停 / 编辑序列 / 复制）。
- **FR-WF-08** 与行为系统关系：工作流是「带序列与时序的扩展编排」，FR-BEH 仍是单层即时行为；工作流可调用 BEH 动作，不重复定义。
