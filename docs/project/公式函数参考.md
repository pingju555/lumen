## 2. 变量与数据源体系

公式是"只读变量"的求值；本项目把变量分为三类，全部映射到 Windows 原生数据源。

### 2.1 全局变量 `gv`（用户定义，跨层共享）
- **定义**：用户在设置面板创建，存于 JSON 全局变量表（`globals_list` 模型：`{title, type, value, index}`）。
- **类型**（KLWP 实测）：`NUMBER` `TEXT` `COLOR` `FONT` `SWITCH` `TIME`。`value` 自身可为公式（如 `gv(accent)` 取 Windows 强调色）。
- **读取**：`gv(name)`；列表型加 `[index]`。
- **作用域**：**全局**——跨所有层/原子共享，是"主题色/用户配置/状态开关"的载体。
- **桌面映射**：JSON 文件 + 内存表，修改即标记依赖原子重算。

### 2.2 内置数据源（函数即变量读取）
每个"系统函数"本质是可计算变量，数据源全本地：

| 数据源 | Kustom 函数 | 桌面原生 API（v1 实现） |
|--------|-------------|--------------------------|
| 系统时钟 | `df` `tf` `ts` `tu` `tz` | `GetLocalTime` / `GetSystemTime` / `std::time` |
| CPU/内存/网速/磁盘/进程 | `si(...)` 扩展字段 | `PDH` / `GlobalMemoryStatusEx` / `GetIfTable` / `GetDiskFreeSpace` / `PSAPI` |
| 电量 | `bi` | `GetSystemPowerStatus` |
| 强调色/暗色模式 | `si(sysca/syscn/darkmode)` | `DwmGetColorizationColor` / Registry |
| 屏幕/分辨率/DPI | `si(rwidth/rheight/density)` | `GetSystemMetrics` / `GetDpiForSystem` |
| 媒体 | `mi` `mu` | `Windows.Media.Control` (SMTC) |
| 已装应用 | `ai` `an` `ag` | 开始菜单 + 注册表枚举 |
| 天气/位置/健身 | `wi` `wf` `wg` `li` `fd` | v1 预留（无本地源） |

### 2.3 上下文变量（模块/层作用域）
KLWP 提供 `$si(mindex)$`（当前模块序号）、`$si(rwidth)$`（根宽）等——本项目对应：
- `si(mindex)` → 网格单元序号（多实例原子区分）。
- `si(rwidth/rheight)` → 覆盖窗口客户区尺寸（响应式布局）。
- `gv(index)` → 列表型全局取项（与 `fl` 类似）。

### 2.4 公式三元组衔接
每个原子属性三选一（见 `Kustom 原子模型.md`）：
```
静态值  ─┐
全局变量 ─┤→ internal_toggles 决定走哪条
公式    ─┘   （公式优先 → gv → 静态）
```
公式结果驱动：文本原子内容、形状尺寸/颜色、进度百分比、图标选择（`if(wi(code)=CLEAR, Sun, Moon)` 范式）、容器显隐。

## 3. 运算符与文法（复用 `公式引擎设计.md` §3）

```
formula := '$' expr '$'          # 可嵌套、可出现在静态文本中
expr    := if_expr | logic_expr
if_expr := 'if' '(' cond ',' expr [',' expr] ')'
cond    := expr cmp expr         # = > >= < <= != ~=
logic   := add (('&'|'|') add)*
add     := mul (('+'|'-') mul)*
mul     := unary (('*'|'/') unary)*
unary   := '-' unary | primary
primary := NUMBER | STRING | COLOR | IDENT '(' [args] ')' | IDENT | '(' expr ')'
IDENT   := [a-z]{2,3}            # 2–3 字母函数/变量名
```
- 字符串拼接 `+`：`if(bi(level)=100,"满电","剩 "+bi(level)+"%")`。
- `~=` 正则匹配：`df(MMMM) ~= "a"`。
- 函数可嵌套、可出现在任意参数位。

## 4. v1 落地范围（函数取舍已定稿 + 桌面相关性复核）

> **筛选总原则（桌面相关性）**：仅保留"存在 Windows 原生数据源（或 v1.x 计划接入）"的函数。移动端独有、桌面**无任何对应源**的函数 → D档 **删除**（不进词表，求值器不注册）。
> 原取舍维度（实现成本 vs 数据源依赖）仍然成立；本次复核（由"桌面无电池"触发）新增"桌面无此硬件/数据"硬删除项。
> 用户拍板：B档纯计算全部并入 v1；`sh`/`wg` 维持 v1.x。

### A档 · v1 核心（本地数据源，必做）— 11 个
```
df  tf  si  bi  gv  if  tc  mi  mu  ai  an
```
- `df`/`tf` → 时钟/日历原子。
- `si`(扩展桌面字段) → 系统监控原子（CPU/内存/网速/磁盘/进程/屏幕/强调色）。
- `bi` → 电量（见 §4.1 桌面电池策略）。
- `gv` → 全局变量（主题色/配置）。
- `if`/`tc` → 条件显隐、文本截断。
- `mi`/`mu` → 媒体原子（SMTC）。
- `ai`/`an` → 启动坞（已装程序枚举）。

### B档 · 纯计算，近零成本（已并入 v1）— 8 个
```
ts  tu  tz  uc  re  fl  dp  rng
```
- 无外部数据源依赖，纯 C# 实现（`DateTime`/`System.Text.RegularExpressions`/`Random`/DPI 换算）。
- 显著增强表达力：文本处理(`uc`/`re`)、列表(`fl`)、随机特效(`rng`)、响应式布局(`dp`)、时间戳/UTC/时区(`ts`/`tu`/`tz`)。

> **v1 必做集 = A档 11 + B档 8 = 19 个函数。**

### C档 · v1.x 预留（有 Windows 源，待接入）— stub 占位
```
wi  wf  ca  cc  nc  ns  ni  bt  ag  fs  sh  br
lo  kg  mp  rg  fm  ge  kp  kv  dn  ds  fe  ff  cm
```
- `sh`(shell 执行)：待专门的**安全沙箱**设计后再放开。
- 天气(`wi`/`wf`)、日历(`ca`/`cc`)、通知(`nc`/`ns`/`ni`)、蓝牙(`bt`)：v1.x 逐步接 Windows 原生源。
- 已落地（移出预留）：`wg`(RSS) / `ce`(颜色) / `bp`(调色板) 均已在 v1 实现。
- `fs`(磁盘空间)：`GetDiskFreeSpace` 本地可得，参数明确后可升 A/B。
- `kp`(快捷键状态)、`cm`(强调色)：桌面相关，可升 A/B。
- §1.14 其余待定参数函数随参数补全后归入本档或降级。

### D档 · 删除（无 Windows 桌面数据源 / 移动端独有 / 手机特性）
```
li  fd  ci  vb  wv
```
- `li`(位置/GPS)：桌面无定位源 → 删。
- `fd`(健身/Google Fit)：无对应源 → 删。
- `ci`(联系人)：隐私 + 无源 → 删。
- `vb`(振动)：手机独有 → 删。
- `wv`(音频波形)：SMTC 不暴露波形，桌面无源 → 删（由 C档降级，与原 §1.14 ❌ 标记一致）。

### §4.1 桌面电池策略（`bi` 去留）
- **结论：保留 `bi`，不删。** 理由：Windows 含笔记本/平板，电池是合法 Windows 数据源（`GetSystemPowerStatus`）；原 A档 已含。
- **无电池回退**：`BatteryProvider` 检测 `BatteryFlag & 128`（无电池）→ `bi(level)` 返回 `Null`，公式回退静态值；或按 UX 约定固定返回 `100`。
- **纯 tower 部署**：若产品明确只跑无电池台式机，`bi` 可整体移入 D档；默认保留以支持笔电。
