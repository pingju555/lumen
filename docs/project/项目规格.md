# 桌面覆盖层 · 项目规格

> 目标：全屏覆盖原生桌面，以网格盛放部件来扩展桌面功能。
> 状态：地基版（已决策项已固化，待议项列为决策清单）。

---

## 1. 项目定位

- **本质**：一个常驻的"桌面替换层"，以全屏覆盖窗口实现。
- **原生桌面处理**：原生桌面图标 + 壁纸 **不可见、不可交互**。覆盖层接管视觉与输入。
- **功能扩展方式**：覆盖层上以 **基于 Px 的细致画布** 自由放置 **原子**（最小可渲染单元，像素级定位）；原子打包为 **小组件**（可复用组合）后，可纳入 **网格** 整齐排布。两层并存：Px 画布（原子自由定位）+ 网格（容纳小组件）。
- **与本地LLM项目关系**：可选。部件未来可接入本地LLM项目的对话入口（节点工作流画布不在此项目内）。

## 3. 任务栏约束（已决策，2026-07-15 修订）

**用户确认：任务栏由系统自动隐藏（Auto-hide），覆盖层不预留、不接管其显隐。** 与早期"预留任务栏区域"方案相反。

覆盖层处理：
- 铺 `SystemParameters.WorkArea`（自动隐藏场景下即全视觉区域），**不单独预留任务栏尺寸**。
- 覆盖层 `Topmost=False`（非全局置顶），保证任务栏弹出时浮于覆盖层之上，覆盖层局部被遮挡可接受。
- 鼠标贴近屏幕下边缘 → 系统唤起任务栏 → 浮于覆盖层之上；移开 → 系统自动隐藏。覆盖层不拦截该边缘触发。
- 不做自建任务栏代偿（无开始菜单 / 托盘 / 通知中心重建需求）。

> 早期"预留任务栏区域"方案已废弃（用户 2026-07-15 修改定稿）。

## 4. 架构草案

### 4.1 多层模型（核心抽象）
覆盖窗口内不再固化"背景层+网格层"，而是 **一组可堆叠的层（Layer）**。每层是独立可配置对象，按 z-order 自上而下合成，以支持进阶操作（独立显隐、混合、模糊、标注、热区等）。

**层类型（预设三类 + 可扩展）：**
- **壁纸层**：仅含一个背景（图片 / 纯色 / 线性·径向渐变 / 毛玻璃模糊），替代原生壁纸，位于最底层。
- **网格化层**：网格格点（cell）用于盛放网格部件（小组件）；原子不进网格、进画布。
- **原子化 Px 画布层**：Px 画布，原子以 (x,y,w,h) 亚像素自由定位。
- 效果层 / 交互层等后续可扩展（进阶层，可选）。

**每层统一可配置项：**
| 属性 | 说明 |
|------|------|
| 启用 / 禁用 | 层总开关 |
| 不透明度 | 0–100% |
| 混合模式 | normal / multiply / screen / overlay … |
| z-order | 层叠顺序 |
| 输入模式 | 拦截 / 穿透 / 仅部件区域可交互 |
| 绑定范围 | 全局 / 指定页面 |
| 显隐规则 | 常显 / 按页面 / 按快捷键 / 自动 |
| 视觉样式 | 背景、边框、圆角、模糊 |
| 更新策略 | 静态 / 定时 / 事件驱动 |

**布局模型**：详见 `功能需求.md` §3.0。要点——原子走 Px 画布（绝对 px 自由定位），小组件走网格（cell）；一个内容层取 Canvas / Grid 其一，同屏要两者并存则建两个层（画布层 + 网格层）按 z-order 合成；Widget 内部仍由原子 px 子画布渲染。

### 4.2 窗口与合成
- 覆盖窗口：全屏、**非全局置顶（Topmost=False）**、无边框、按各层输入模式合成输入。
- 各层按 z-order 从下到上合成。
- 任务栏：系统自动隐藏，覆盖层铺满工作区不预留，弹出浮于其上（见 §3）。
- 多显示器：多开模型，每实例覆盖一块屏（见 FR-COV-06/07 与 §7）。

### 4.3 层级映射（层实例）
1. **覆盖窗口**：全屏、**非全局置顶（Topmost=False）**、`WS_EX_TOOLWINDOW`/无边框、鼠标键盘拦截（不向原生桌面穿透）。
2. **背景层**：自定义渲染（图片缩放/纯色/毛玻璃模糊）。
3. **网格/画布层**：Px 画布承载原子（自由定位）；网格子系统承载**小组件**（原子组合），小组件锚定到单元。
4. **页面管理**：多套网格布局，切换动画/指示器。
5. **部件系统**：统一部件接口（生命周期、配置、渲染、刷新）。
6. **控制层**：显隐切换、设置面板、**层管理（增删 / 排序 / 配置）**。

### 4.4 进阶操作：动画 / 触发器 / 按钮行为
多层模型的目标是用层组合出进阶交互。每层（及层内元素）可绑定：
- **动画**：进入 / 退出 / 循环动画（淡入、滑动、缩放、旋转等），参数含时长、缓动、延迟。
- **触发器**：事件源（显示/隐藏层、点击、悬停、空闲 N 秒、定时、系统事件、页面切换）→ 触发一组动作。
- **按钮行为**：层/部件上的按钮可配置动作（切换层显隐、切页、运行程序、发快捷键、触发动画、开 URL、接本地LLM 等）。

详见 `功能需求.md` §1.7。

### 4.5 层预设快速添加
为降低上手成本，层添加采用**固定预设**而非每次手动配参：
- 预设模板预填 9 项通用属性 + 相关动画/触发器默认值。
- 预设类型：壁纸层 / 网格化层 / 原子化 Px 画布层（可扩展效果 / 交互等）。
- 添加后可在设置面板微调；部件添加方式后续定义（本期不做）。

### 部件接口（草案）
```
Widget {
  id, gridPos {row,col,rowSpan,colSpan}
  init(config)
  render()        // 返回 UI 节点
  update()        // 定时/事件刷新
  onConfigChange()
  destroy()
}
```

## 6. 技术栈（已定稿：C# / WPF）

> 完整选型与守护进程设计见 **`技术栈选型.md`**（2026-07-14 由 Rust+Tauri+Slint 切换至 C#/WPF 定稿）。

### 定稿：C# / WPF（.NET 8，无 WebView2）
- 覆盖层需直接与 Win32 窗口层级 / 样式 / 输入拦截打交道；Tauri + Slint 框架抽象与 Win32 定位反复冲突、编译-修复循环过多，故切换。
- WPF 原生持有 `HWND`，XAML 声明式描述原子 / 层，开发可控性更优；.NET 8 SDK 已就绪。
- 轻量、贴合"原生桌面"目标、零 WebView2 运行时负担（v1 无网页部件）。
- 系统指标（CPU/内存/网速）用 `PerformanceCounter`(PDH)/WMI；启动其他程序用 `Process.Start`；媒体用 WinRT SMTC。
- 全局快捷键（显隐）用 `RegisterHotKey` P/Invoke；托盘用 `Hardcodet.NotifyIcon.Wpf`；单实例用命名 `Mutex`——覆盖层必需。

### 守护进程（防误杀，已定）
- **单一 exe 双模式**：`overlay.exe` 正常=UI；`overlay.exe --guardian`=无窗口看门狗。分发=一个文件。
- **用户态互相看门**：两进程互持句柄 + 命名管道心跳，任一被杀秒级重拉；命名事件 `overlay-clean-shutdown` 标记主动退出（不重启）。
- **崩溃兜底**：`RegisterApplicationRestart`。
- **开机自启**：注册表 `HKCU\...\Run`（零安装）。
- 维持无 WebView2（v1 无网页部件）；若未来需网页部件 → 用 **WPF `WebView2` 控件**（`Microsoft.UI.WebView2`），不切 Tauri/Electron（见 `技术栈选型.md` §5.2）。

### 切换条件（仅未来扩展）
- 若后续引入 **网页/iframe 部件** → 采用 **WPF `WebView2` 控件**（`Microsoft.UI.WebView2`），届时守护逻辑不变。
- 纯自绘（Skia/Direct2D 自定义渲染面）：控制最强但开发量最大，不推荐。
