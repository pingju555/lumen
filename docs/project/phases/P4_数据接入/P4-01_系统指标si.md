# P4-01 系统指标 si()

> Phase：P4 · 数据接入（si / mi / mu / ai / an）
> 上游：`v1开发计划.md` P4、`功能需求.md` §6 系统监控、FR-SYS-*`、`技术栈选型.md` §6（系统指标）
> 关联：`P4-04_公式Provider闭环.md`、`P2-03_公式引擎.md`、`P2-05_增量重算与容错.md`

## 目标
用 `System.Diagnostics.PerformanceCounter`（PDH）采集系统指标，经 `si()` 函数暴露给公式引擎：CPU 占用、内存、网速（上行 / 下行）、磁盘读写、进程数等。

## 范围
**包含**
- `SiProvider : IDataProvider` 实现，承接 `si(cpu)` / `si(mem)` / `si(net.down)` / `si(net.up)` / `si(disk.r)` / `si(disk.w)` / `si(proc)` 等。
- 采样：后台 `Timer` 周期读 PDH（如 1s），缓存最新值；公式读取缓存（不每次重采样）。
- 网速 / 磁盘为**速率**：需两次采样差值 / 间隔计算（v1 精度按 `技术栈选型.md` §6.2 留精化余地）。

**不含**：精度精化（PDH 多实例聚合，留 v1.x）；历史曲线存储（v1 仅当前值）。

## 关键设计
### PDH 采样
```csharp
class SiProvider : IDataProvider {
    PerformanceCounter _cpu = new("Processor","% Processor Time","_Total");
    PerformanceCounter _mem = new("Memory","% Committed Bytes In Use");
    // 网速：Interface 类别，需选活跃网卡
    public Value Get(string name, Value[] args) {
        return name switch {
            "cpu" => _cpu.NextValue(),
            "mem" => _mem.NextValue(),
            "net.down" => SampleNetwork(false),
            "net.up"   => SampleNetwork(true),
            _ => Value.Null
        };
    }
}
```
- `NextValue()` 首次返回 0，需预热（启动时先调一次丢弃）。

### 缓存 + 周期
```csharp
Timer _tick = new(1000);  _tick.Elapsed += (_,_) => RefreshAll();  // 更新缓存
```
- 公式 `si(...)` 读缓存；`P2-05` 脏调度在每秒 tick 后标记相关原子重算。

## 技术选型
`System.Diagnostics.PerformanceCounter`（PDH）、`System.Timers.Timer`（`技术栈选型.md` §6 系统指标行）。无第三方。

## FR 映射
FR-SYS-*（系统监控部件）；si 函数（`公式函数参考.md`）。

## 验收
`$si(cpu)$` / `$si(mem)$` / `$si(net.down)$` 实时更新；网速为合理速率值（非累计字节）。

## 依赖与顺序
- 依赖：`P2-03`(`IDataProvider` 接口)、`P2-05`(脏调度)。
- 被依赖：`P4-04`(聚合进引擎)、P6-03 打磨（NFR 抽检）。

## 风险 / 开放
- 多网卡选择：需识别"活跃"网卡（非 VPN / 环回），v1 取首个非环回或用户可选。
- PDH 本地化计数器名：非英文系统计数器名不同，需按 `CultureInfo` 或索引取（注意国际化）。
- CPU 采样首次 0 的预热处理。
