# P5-02 触发器

> Phase：P5 · 行为系统（动画 / 触发器 / 按钮）
> 上游：`v1开发计划.md` P5、`功能需求.md` §1.7 行为系统、FR-BEH-*`、`技术栈选型.md` §6（触发器）
> 关联：`P5-01_动画.md`、`P5-03_按钮行为.md`、`P4-02_媒体mi_mu.md`、`P2-05_增量重算与容错.md`

## 目标
实现多种触发器，满足条件时驱动动作（播放动画 / 切层 / 切页 / 运行程序等）：显示层 / 点击 / 悬停 / 空闲(`GetLastInputInfo`) / 定时 / 系统事件（锁屏 · 电量低 · 网络变更）。

## 范围
**包含**
- `Trigger` 类型枚举 + 监听机制：
  - **显示层**：特定层显隐切换。
  - **点击 / 悬停**：原子 `UIElement` 的 `MouseDown` / `MouseEnter`。
  - **空闲**：`GetLastInputInfo` 轮询，超过 N 秒触发。
  - **定时**：`DispatcherTimer` 到点（如每分钟）。
  - **系统事件**：锁屏（`WTSQuerySessionNotification` / `SystemEvents`）、电量低（`WM_POWERBROADCAST`）、网络变更（`NetworkInformation`）。
- 触发器 → 动作绑定（动作见 `P5-03`）。

**不含**：复杂条件组合表达式（v1 单触发→动作；多条件留 v2）；用户脚本 DSL（v1 内置类型）。

## 关键设计
### 模型
```csharp
enum TriggerKind { ShowLayer, Click, Hover, Idle, Timer, Lock, LowBattery, Network }
class Trigger {
    public TriggerKind Kind;
    public Dictionary<string,object> Params;   // 如 Idle.Seconds, Timer.Interval
    public List<Action> Actions;              // 触发后执行
}
class TriggerEngine {
    void Register(Trigger t);
    // 各 Kind 监听：事件订阅 / 轮询
}
```

### 监听要点
```csharp
// 空闲
uint lastInput = 0; GetLastInputInfo(ref lastInput);
if (Environment.TickCount - lastInput > idleMs) Fire(TriggerKind.Idle);

// 系统事件
SystemEvents.SessionSwitch += (_, e) => { if (e.Reason==SessionSwitchReason.SessionLock) Fire(Lock); };
// 电量：RegisterPowerSettingNotification(GUID_POWER_SAVING_STATUS) 或 WMI
```

## 技术选型
Win32 `GetLastInputInfo` / `SystemEvents` / `NetworkInformation`（`技术栈选型.md` §6 触发器行）。无第三方。

## FR 映射
FR-BEH-（触发器相关，§1.7 行为系统）。

## 验收
空闲 N 秒触发显隐；点击原子触发动画；锁屏触发某层显隐；定时每分钟执行动作；网络断开触发提示层。

## 依赖与顺序
- 依赖：`P2-05`(脏/刷新)、`P4-02`(媒体事件参考)、`P5-03`(动作执行)。
- 被依赖：P6-02 设置面板配置触发器、便签/工作流（P6 / 版本排期 v1.2）。

## 风险 / 开放
- 空闲轮询间隔：建议 1s 精度足够，避免高频。
- 系统事件权限：电量低需注册电源通知，部分需 manifest；网络变更用 `NetworkInformation.NetworkStatusChanged`（WinRT 或 Win32）。
- 触发器与动画/降频协同：触发大批动画时经 `P2-05` 合并。
