# P3-02 小组件

> Phase：P3 · 网格 + 小组件 + 页面
> 上游：`v1开发计划.md` P3、`功能需求.md` FR-WIDGET-01~07、`技术栈选型.md` §6（小组件）
> 关联：`P2-01_六类原子.md`、`P3-01_网格.md`、`P3-03_页面.md`、`P2-06_持久化.md`

## 目标
将原子树（一组原子 + 布局）打包为**小组件**，注册到 `WIDGET_REGISTRY`，可拖入网格 cell、整体移动 / resize / 显隐、嵌套、属性继承，并入库复用。

## 范围
**包含**
- `Widget`：`Atom[]` 子树 + 内部布局（相对坐标）+ 占用 cell 区间。
- `WIDGET_REGISTRY`：`Dictionary<string, Widget>` 增删改查（内置 + 用户）。
- 拖入 grid cell：从库拖到 `P3-01` cell，实例化原子树挂 `GridLayer`。
- 整体操作：移动（整体平移子树 bounds）/ resize（按 cell 缩放子树）/ 显隐。
- 嵌套：小组件可含子小组件（递归实例化）。
- 属性继承：父 widget 的 gv / 变量向子树暴露（v1 轻量：共享 gv 表）。

**不含**：小组件市场 / 在线分享（v1 本地库）；复杂布局约束求解（v1 相对坐标 + cell 对齐）。

## 关键设计
### 模型
```csharp
class Widget {
    public string Id;
    public List<Atom> Atoms = new();        // 原子树（含 Container 嵌套）
    public Rect CellSpan;                   // 占用 cell 区间
}
static class WIDGET_REGISTRY {
    Dictionary<string, Widget> _lib = new();
    public void Add(Widget w);
    public Widget? Get(string id);
    public Widget Instantiate(string id);   // 深拷贝 → 挂画布
}
```

### 入库 / 复用
- 选中画布上一组原子 → "打包为小组件" → 存入 `WIDGET_REGISTRY`（内存 + 落盘 `P2-06`）。
- 拖拽库项到 cell → `Instantiate` 深拷贝 → 吸附到 cell → 加入 `GridLayer`。

### 整体操作
```csharp
class WidgetInstance {
    void Move(Vector d);                    // 全部子原子 Bounds 平移
    void ResizeToCells(int cols, int rows); // 按 cell 缩放子树（保持相对比例）
    void SetVisible(bool v);
}
```

## 技术选型
C# 集合 + 深拷贝（序列化往返或手动 clone）（`技术栈选型.md` §6 小组件行）。无第三方。

## FR 映射
FR-WIDGET-01~07（打包 / 入库 / 拖入 / 整体操作 / 嵌套 / 继承）。

## 验收
打包「时钟小组件」（Text+Shape）拖入网格 cell；整体移动 / 2×2 resize 对齐；入库后可再次拖出复用；嵌套小组件正确展开。

## 依赖与顺序
- 依赖：`P2-01`(原子树)、`P3-01`(cell 吸附)、`P2-06`(widget DTO 持久化)、`P2-04`(gv 共享)。
- 被依赖：P3-03 页面切换含小组件、P3-04 预设库的"用户自定义预设"复用 widget。

## 风险 / 开放
- 深拷贝成本：大 widget 序列化往返可行，需避免循环引用（Container 自包含子树）。
- 属性继承深度：v1 仅共享 gv，不做局部作用域变量（留 v2）。
