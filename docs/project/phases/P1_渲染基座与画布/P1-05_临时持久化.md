# P1-05 临时持久化（单原子）

> Phase：P1 · 渲染基座 + 多层 + Px 画布 + 单原子
> 上游：`v1开发计划.md` P1（临时 JSON 存一个原子）、`技术栈选型.md` §6（持久化）、`功能需求.md` §3.0（每屏×每层）
> 关联：`P1-04_Px画布与Text原子.md`、`P2-06_持久化.md`（全量升级）

## 目标
用 `System.Text.Json` 把单个原子的 `x/y/w/h` + 静态文本落盘，验证"刷新后保留"链路。这是 P2-06 全量持久化的雏形，schema 需向前兼容。

## 范围
**包含**
- 原子 `Bounds`(Rect) + 静态文本序列化为 JSON。
- 配置目录：`%LocalAppData%/Overlay/config.json`（v1 单屏单文件，多屏×每层结构留 P2-06）。
- 启动加载 → 重建原子 → 挂画布。

**不含**：六类原子全字段、层配置、gv、页面、行为（P2/P3/P6 各自补全）；schema 版本迁移（P2-06 / `技术栈选型.md` §5.1）。

## 关键设计
### 临时 schema（v1 雏形）
```json
{
  "version": 1,
  "atoms": [
    { "type": "Text", "x": 100, "y": 100, "w": 200, "h": 40, "text": "Hello" }
  ]
}
```

### 读写
```csharp
record AtomDto(string Type, double X, double Y, double W, double H, string Text);
class ConfigStore {
    const string Path = "%LocalAppData%/Overlay/config.json";
    void Save(IEnumerable<Atom> atoms);
    IEnumerable<AtomDto> Load();
}
```
- `System.Text.Json`（源生，零依赖）；可空处理 + 缺省兜底，避免坏文件崩。

## 技术选型
`System.Text.Json` + `%APPDATA%`(`Environment.GetFolderPath`) 用户配置目录（`技术栈选型.md` §6 持久化行）。非注册表。

## FR 映射
功能需求 §3.0（持久化结构前置）、FR-SET-08（每屏×每层，此处单屏雏形）。

## 验收
放 Text 并拖动 → 退出重启 → 原子位置 / 文本一致恢复。

## 依赖与顺序
- 依赖：`P1-04`(原子实例)。
- 被依赖：P2-06 升级为全量 schema（atoms + layers + gv + 每屏×每层）。

## 风险 / 开放
- 临时 schema 会被 P2-06 替换，需保证 `version` 字段存在以便迁移（即便 v1 即 v1）。
- 多屏×每层结构（FR-COV-06/07）在 P2-06 引入，此处先单文件单屏。
