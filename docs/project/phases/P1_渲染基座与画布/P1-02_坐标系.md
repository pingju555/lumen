# P1-02 坐标系

> Phase：P1 · 渲染基座 + 多层 + Px 画布 + 单原子
> 上游：`v1开发计划.md` P1、`功能需求.md` §3.0（三套坐标系）
> 关联：`P1-01_多层模型.md`、`P1-04_Px画布与Text原子.md`、`P3-01_网格.md`

## 目标
定义并打通三套坐标系及其换算：
- **Screen**：工作区像素，原点左上（扣任务栏预留，但 Auto-hide 下即满屏），单位 px。
- **Canvas(px)**：原子在所属层内的自由像素坐标，原点层左上。
- **Grid(cell)**：网格单元坐标，基于 gcd 间距推导（P3 细化，此处先定义接口）。

## 范围
**包含**
- 三套坐标的原点 / 单位定义与数据结构。
- `cell ↔ px` 换算接口（gridSize 由 P3 网格提供，此处留注入点）。
- Canvas(px) 与 Screen 的偏移换算（层在屏内偏移）。

**不含**：gcd 网格具体数学（P3-01）；DPI 换算由 WPF 布局自动处理（`Per-Monitor v2`，`技术栈选型.md` §5.1）。

## 关键设计
### 定义
```csharp
struct ScreenPoint  { double X, Y; }   // 工作区 px
struct CanvasPoint  { double X, Y; }   // 层内 px
struct GridCell     { int Col, Row; }  // 网格单元
```

### 换算
```csharp
static class Coord {
    // 层在屏内偏移 + 层内 px → 屏 px
    ScreenPoint CanvasToScreen(Layer layer, CanvasPoint p);
    // 网格单元 + 当前 gridSize → 层内 px
    CanvasPoint GridToCanvas(GridCell c, double gridSize);
    CanvasPoint ScreenToCanvas(Layer layer, ScreenPoint p);
}
```
- v1 单层覆盖全工作区 → 层偏移为 (0,0)，`CanvasToScreen` 近似恒等（预留多页/多屏偏移扩展）。
- `gridSize` 由 `P3-01_网格.md` 的 gcd 四档 {20,40,60,120} 提供，P1 先以常量占位。

## 技术选型
纯 C# 结构 + 转换函数；WPF 布局负责最终 DPI/设备像素映射。无外部依赖。

## FR 映射
功能需求 §3.0（Screen / Canvas / Grid 三坐标系）。

## 验收
原子以 Canvas(px) 存放、网格以 cell 存放；`GridToCanvas` 在 1920×1080 四档下换算正确；Canvas↔Screen 在单屏偏移 (0,0) 下一致。

## 依赖与顺序
- 依赖：`P0-01_覆盖窗口.md`（工作区矩形）、`P1-01_多层模型.md`（层偏移）。
- 被依赖：P1-04 原子定位、P3-01 网格吸附、P2 持久化坐标存盘。

## 风险 / 开放
- 多屏（v2）下 Screen 需绑 `Screen.Id`，P1 先假设单屏。
- 网格吸附精度与 canvas 浮点一致性需在 P3 联调时校验。
