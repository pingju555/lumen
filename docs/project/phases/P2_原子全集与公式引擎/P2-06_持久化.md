# P2-06 持久化（全量 schema v1）

> Phase：P2 · 原子全集 + 属性三元组 + 公式引擎
> 上游：`v1开发计划.md` P2、`功能需求.md` §3.0（每屏×每层）、FR-COV-06/07、FR-SET-08、`技术栈选型.md` §6（持久化）+ §5.1（schema 版本）
> 关联：`P1-05_临时持久化.md`（升级自此）、`P2-02_属性三元组.md`、`P2-04_全局变量gv.md`、`P6-03_持久化闭环.md`

## 目标
将 **atom（x/y/w/h/z + 属性三元组）+ layer 配置 + gv** 全量序列化为 JSON，并升级 `P1-05` 临时 schema 为 **schema v1**，结构支持「每屏 × 每层」。

## 范围
**包含**
- 原子 DTO：类型 / Bounds / ZIndex / 各属性三元组（静态 / gv 引用 / 公式串）。
- 层 DTO：类型（壁纸 / 网格 / 画布）/ ZIndex / Opacity / Enabled。
- gv DTO：名 / 类型 / 值。
- 结构：`{ version:1, screens:[{ id, layers:[{ kind, atoms:[...], gv:[...] }] }] }`（v1 单屏单实例，结构预留多屏）。
- schema 版本字段 + 向前兼容迁移钩子（破坏性变更仅 v2.0，`技术栈选型.md` §5.1）。

**不含**：行为 / 小组件数据 / 文件速览 / 工作流（P5/P3/P6 各自补全 DTO）；导入导出 UI（P6-02）。

## 关键设计
### schema v1
```json
{
  "version": 1,
  "screens": [
    { "id": "DISPLAY1",
      "layers": [
        { "kind": "Canvas", "z": 1, "opacity": 1, "enabled": true,
          "atoms": [
            { "type": "Text", "x": 100, "y": 100, "w": 200, "h": 40, "z": 0,
              "props": { "text": { "mode": "formula", "value": "$df(hh:mm:ss)$" } } }
          ],
          "gv": [ { "name": "clock_fmt", "type": "text", "value": "HH:mm" } ]
        }
      ]
    }
  ]
}
```

### 读写
```csharp
class ConfigStore {
    const int Version = 1;
    void Save(Document doc);             // Document = screens/layers/atoms/gv 内存模型
    Document Load();                     // 坏文件 → 默认空文档 + 日志，不崩
    Document Migrate(JObject raw);       // 按 raw.version 迁移到当前
}
```
- `System.Text.Json`；DTO 用 `record` / `class` + `JsonSerializerOptions { PropertyNameCaseInsensitive = true }`。
- 坏文件 / 缺字段 → 兜底默认 + `ILogger` 警告（`技术栈选型.md` §5 日志项）。

## 技术选型
`System.Text.Json` + `%LocalAppData%/Overlay/config.json`（`技术栈选型.md` §6 持久化行）。零依赖。
