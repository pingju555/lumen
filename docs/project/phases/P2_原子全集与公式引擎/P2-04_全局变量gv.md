# P2-04 全局变量 gv

> Phase：P2 · 原子全集 + 属性三元组 + 公式引擎
> 上游：`v1开发计划.md` P2、`功能需求.md` gv（跨层共享）、`技术栈选型.md` §6（gv）
> 关联：`P2-02_属性三元组.md`、`P2-03_公式引擎.md`、`P2-05_增量重算与容错.md`

## 目标
实现类型化全局变量 `gv`，跨层共享，改值即触发增量重算。公式 `/ gv 引用` 通过 `EvalContext` 读取。

## 范围
**包含**
- 类型：`NUMBER` / `TEXT` / `COLOR` / `FONT` / `LIST`（至少前四类）。
- `GvStore`：`Dictionary<string, TypedValue>`，线程安全（UI 线程为主，但 gv 可从设置面板 / 行为改）。
- 变更通知：`INotifyPropertyChanged` / 事件，驱动 `P2-05` 脏标记重算。
- 持久化：gv 随配置存盘（接 `P2-06`）。

**不含**：gv 的 UI 编辑（P6 设置面板）；gv 与外部数据源绑定（v1 为本地值）。

## 关键设计
### 类型化值
```csharp
enum GvType { Number, Text, Color, Font }
class TypedValue {
    public GvType Type;
    public object Raw;            // double / string / Color / FontFamily
}
class GvStore : INotifyPropertyChanged {
    Dictionary<string, TypedValue> _store = new();
    public TypedValue Get(string name);
    public void Set(string name, TypedValue v);   // 触发 PropertyChanged
    public event PropertyChangedEventHandler? Changed;
}
```

### 接入公式
```csharp
class EvalContext {
    public GvStore Gv;
    public Value ResolveGv(string name) => Gv.Get(name).ToValue();
}
// 公式 $gv(clock_fmt)$ → EvalContext.ResolveGv("clock_fmt")
```

### 增量触发
- `GvStore.Changed` → 标记所有引用该 gv 的原子为脏 → `P2-05` 重算。
- gv 名可加作用域（v1 全局；多页/多屏作用域留 v2）。

## 技术选型
C# `Dictionary` + `INotifyPropertyChanged`（`技术栈选型.md` §6 gv 行）。无依赖。

## FR 映射
功能需求 gv（跨层共享全局变量）；属性三元组 Global 模式（P2-02）。

## 验收
`$gv(x)$` 在多个层原子引用同一 gv；改 gv 值 → 所有引用原子即时刷新；gv 存盘重开一致。

## 依赖与顺序
- 依赖：`P2-03`(EvalContext 读取)。
- 被依赖：P2-02 三元组 GvRef、P2-05 脏标记、P6 设置面板编辑 gv。

## 风险 / 开放
- gv 循环（公式改 gv、gv 触发公式）需防自触发死循环（与 P2-05 容错协同）。
- 类型强约束 vs 宽松：建议"赋值强类型、读取宽松（失败回退）"。
