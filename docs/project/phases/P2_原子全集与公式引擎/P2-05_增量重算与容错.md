# P2-05 增量重算与容错

> Phase：P2 · 原子全集 + 属性三元组 + 公式引擎
> 上游：`v1开发计划.md` P2（脏标记增量重算；求值错误回退静态值不崩）、NFR-07（降频）、`技术栈选型.md` §6
> 关联：`P2-02_属性三元组.md`、`P2-03_公式引擎.md`、`P2-04_全局变量gv.md`、`P0-01`(降频)

## 目标
- **增量重算**：仅对"脏"原子重新求值 / 渲染，避免全量刷新（守 NFR 性能）。
- **容错**：公式 / gv 求值异常时回退静态值（或上一次有效值），绝不崩溃。
- **降频**：失焦 / 空闲时停绘或降频（`DispatcherTimer` 动态 Interval，见 `技术栈选型.md` §6 渲染降频）。

## 范围
**包含**
- 脏标记：原子属性变更 / gv 变更 / 定时器（df 每秒）触发对应原子脏。
- 调度器：`DirtyScheduler` 收集脏原子 → 批量重算（同帧合并）。
- 容错：`try/catch` 包裹求值，异常 → 回退 `LastGoodValue` / 静态默认值。
- 循环检测：依赖图检测 gv↔公式环，断环（最多 N 层展开）。
- 降频：`DispatcherTimer` Interval 随焦点 / 空闲态调整；失焦停 `CompositionTarget.Rendering`。

**不含**：具体公式语义（P2-03）；gv 存储（P2-04）；行为触发器调度（P5，复用脏机制）。

## 关键设计
### 脏标记 + 调度
```csharp
class DirtyScheduler {
    HashSet<Atom> _dirty = new();
    public void MarkDirty(Atom a);
    public void Flush() {                      // 每帧/定时调用
        foreach (var a in _dirty) a.Update();  // 重算+重渲
        _dirty.Clear();
    }
}
// gv.Changed → 标记引用该 gv 的原子；定时器(每秒) → 标记含 df/tf 的原子
```

### 容错
```csharp
Value SafeEval(string expr, EvalContext ctx, Value fallback) {
    try { return ctx.Eval(expr); }
    catch { return fallback; }       // 回退静态/上值，记日志
}
```
- 每个原子缓存 `LastGoodValue`，异常时继续显示旧值（不闪空）。

### 依赖图与循环
- 构建 原子→gv→原子 依赖边；求值时若深度 > 阈值或检测到回边 → 断开并回退。
- 预留 `maxRecursion` 常量（如 32）。

### 降频
- `DispatcherTimer` 默认 1s（df 精度）；窗口失焦 → 停 `Rendering` 或拉长 Interval 至 5–10s。
- NFR-07 空闲 CPU~0：无脏标记时不重算、不重绘。

## 技术选型
WPF `DispatcherTimer` + `CompositionTarget.Rendering`（`技术栈选型.md` §6 渲染降频）；脏集合 `HashSet`；`ILogger` 记录求值异常（`技术栈选型.md` §5 日志）。

## FR 映射
NFR-07（基于焦点的失焦停绘 / 降频）；公式容错（功能需求公式章）。

## 验收
改一个原子属性 → 仅该原子重算（其他不变）；公式写错 → 显示旧值不崩；切走窗口 → CPU 趋零、回来立即恢复。

## 依赖与顺序
- 依赖：`P2-02/03/04`（求值与 gv）。
- 被依赖：P4 数据接入（高频更新走同调度）、P5 行为（触发器改属性走同脏机制）。

## 风险 / 开放
- 脏合并粒度：按帧 Flush 可能积压，需平衡实时性与 CPU（df 要求 1s 精度）。
- 多屏（v2）下每屏独立调度器，需按屏停绘。
