# P2-03 公式引擎

> Phase：P2 · 原子全集 + 属性三元组 + 公式引擎
> 上游：`v1开发计划.md` P2、`公式引擎设计.md`、`公式函数参考.md`、`技术栈选型.md` §6（公式引擎）
> 关联：`P2-02_属性三元组.md`、`P2-04_全局变量gv.md`、`P4-04_公式Provider闭环.md`

## 目标
实现 C# 公式求值器 **五段式**：`Lexer → Parser → AST → FunctionRegistry → DataProvider`。支持 v1 必做 **19 函数** 与运算符（`= > >= < <= != ~=`、`+` 拼接、`~=` 正则）。这是 MVP 的核心，证明"原子 → 公式 → 实时数据"链路。

## 范围
**包含**
- Lexer：切分数字 / 字符串 / 标识符 / 操作符 / 括号 / 逗号。
- Parser：递归下降 → 表达式 AST。
- AST：字面量 / 变量(gv) / 函数调用 / 二元运算 / 一元运算 节点。
- FunctionRegistry：注册 19 函数，分 A 档 11 + B 档 8。
- DataProvider：提供 gv 值 + 系统数据接口（si/mi/mu/ai/an 在 P4 接入，此处预留 `IDataProvider`）。
- 运算符求值（含字符串 `=`/`~=` 正则、数字比较、文本 `+` 拼接）。

**不含**：函数具体语义实现细节（见 `公式函数参考.md`）；系统数据真实来源（P4）；增量调度（P2-05）。

## 关键设计
### 五段式
```
text "$if(bi(level)=100,\"满\",bi(level))$"
  → Lexer   → [IF ( bi ( level ) = 100 , "满" , bi ( level ) )]
  → Parser  → IfEq( Gv/Call(bi,level), 100, "满", Call(bi,level) )
  → AST     → 表达式树
  → FunctionRegistry → 分发 bi/if 实现
  → DataProvider     → bi(level) 取电量
  → 结果 "87" / "满"
```

### 19 函数（v1 必做）
| 档 | 函数 | 说明（详见 `公式函数参考.md`） |
|----|------|------|
| A(11) | `df` `tf` `si` `bi` `gv` `if` `tc` `mi` `mu` `ai` `an` | 日期/时间/系统/电量/全局变量/条件/文本/媒体/音乐/应用/应用名 |
| B(8) | `ts` `tu` `tz` `uc` `re` `fl` `dp` `rng` | 时间戳/Unix时间/时区/大写/正则/浮点/小数位/随机 |

### 接口草图
```csharp
class Evaluator {
    public Value Eval(string expr, EvalContext ctx);   // 入口
}
interface IDataProvider {                               // P4 实现系统数据
    Value Get(string name, params Value[] args);
}
static class FunctionRegistry {
    Dictionary<string, Func<Value[], EvalContext, Value>> _fns;
    void Register(string name, Func<Value[], EvalContext, Value> fn);
}
abstract class AstNode { Value Eval(EvalContext ctx); }
```
- `Value`：C# 类 / 枚举（NUMBER / TEXT / COLOR / BOOL…，见 `公式引擎设计.md` §Value 类型）。
- `EvalContext`：持有 gv 表 + `IDataProvider` + 当前时间等。

## 验收
- `$df(hh:mm:ss)$` 每秒刷新。
- `$if(bi(level)=100,"满",bi(level))$` 正确（电量满显"满"，否则显数值）。
- `$uc("abc")$` → "ABC"；`$"x" + "y"$` → "xy"；`$"ab" ~= "a."$` 正则匹配。
- 求值异常不崩（回退静态，见 P2-05）。
