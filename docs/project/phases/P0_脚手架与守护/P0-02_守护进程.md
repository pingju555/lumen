# P0-02 守护进程（双模式看门狗）

> ⚠️ **状态（2026-07-15 用户决策）**：本模块的**看门狗/守护进程已在 v1 移除**。理由：当前无必要保证"进程级常驻/秒级重拉"，且守护进程会导致退出热键退不掉（UI 退了又被拉起）。改为**单进程直接运行 UI**：`overlay.exe` 启动即覆盖窗口，`Ctrl+Alt+Q` 干净退出即进程退出，无重拉。若未来确需常驻自愈，可据本文档设计恢复，但现不实现、代码已删除（`Core/Guardian.cs`）。
> Phase：P0 · 脚手架
> 上游：`v1开发计划.md` P0、`技术栈选型.md` §3（守护）、`功能需求.md` FR-COV-15/17, §3
> 关联：`P0-01_覆盖窗口.md`

## 目标（移除后现状）
`overlay.exe` **单进程**直接运行覆盖窗口，无守护 / 看门狗：
- 启动：`App.OnStartup` 直接 `new OverlayWindow().Show()`。
- 退出：`Ctrl+Alt+Q`（`WM_HOTKEY`）→ `Application.Current.Shutdown()`，进程退出，不再重生。
- 注册表自启仍保留（`Autostart.SetEnabled(bool)`），由 P6 设置面板开关控制；`--install`/`--uninstall` 仅作部署 / 调试入口。

## 范围
**包含**
- `Main(string[] args)` 解析 `--guardian` 分支。
- UI 启动即 `Process.Start` 自身 `--guardian`。
- 命名管道 `\\.\pipe\overlay-guardian` 心跳；命名事件 `Global\overlay-clean-shutdown` 主动退出不重启。
- 互持 `Process` 句柄 `WaitForExit` 重拉。
- 注册表 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` 自启（带开关可取消）。
- `RegisterApplicationRestart` 崩溃兜底。

**不含**：业务逻辑（窗口 / 原子在其它 Phase）；自启 UI（仅注册表项，托盘在 P6）。

## 关键设计
### 双模式入口
```csharp
static int Main(string[] args) {
    if (args.Contains("--guardian")) return GuardianLoop();
    return UiLoop();   // 启动窗口 + Process.Start(self, "--guardian")
}
```

### 互相看门
- UI 进程持有 guardian `Process`，`WaitForExit` 后重拉（除非 clean-shutdown 事件已置位）。
- guardian 进程持有 UI `Process`，`WaitForExit` 后重拉；UI 亦反向监控 guardian 防单边死。
- 心跳：管道周期写入时间戳，超时（如 5s）判定对方失联即重拉。

### 干净退出
```csharp
var ev = new EventWaitHandle(false, EventResetMode.ManualReset, "Global\\overlay-clean-shutdown");
ev.Set();                 // 置位 → 双方 WaitForExit 后均不再重拉
Application.Current.Shutdown();
```

### 自启
- 安装：`Registry.CurrentUser.OpenSubKey(@"Software\...\Run", true).SetValue("Overlay", exePath)`。
- 卸载：删除该值。
- **开关归属**：开机自启是**设置面板里的一个开关（可开可关）**，由 P6 设置面板调用 `Autostart.SetEnabled(bool)` 控制；`--install`/`--uninstall` 命令行仅作调试 / 部署入口，不面向终端用户。

### 崩溃兜底
- `Application.Restart` + `RegisterApplicationRestart(null, 0)`（需 manifest `restartComment`）。
- 与看门狗双保险：看门狗处理进程级被杀，RegisterApplicationRestart 处理 UI 进程内未捕获异常崩溃。

## 接口草图
```csharp
static class Guardian {
    const string Pipe = @"\\.\pipe\overlay-guardian";
    const string CleanShutdownEvent = "Global\\overlay-clean-shutdown";
    int Loop();                              // --guardian 主循环
    void SpawnAndWatch(string targetArgs);  // 拉起并监控
    bool IsCleanShutdownRequested();
}
static class Autostart {
    void Install(); void Uninstall(); void SetEnabled(bool); bool Enabled { get; }
}
```

## 技术选型
- WPF + `System.Diagnostics.Process` + 命名管道（`NamedPipeServerStream`/`ClientStream`）+ 命名事件（`EventWaitHandle`）+ `RegisterApplicationRestart`（P/Invoke）+ 注册表（`Microsoft.Win32.Registry`）。
- 零第三方依赖，守 NFR-01（<80MB）。

## FR 映射
FR-COV-15（守护 / 自恢复）、FR-COV-17（注册表自启）、`功能需求.md` §3（多屏持久化前置，单屏 v1 即单实例）。

## 验收
双击 → 覆盖层出现；杀 UI 进程 → 看门狗秒级拉起；杀 guardian → UI 重拉；点"退出" → 双方均不重启；自启生效（重启后自动起）；UI 进程内抛未捕获异常 → 系统自动重启。

## 依赖与顺序
- 无前置（P0 起点），但逻辑上晚于 `P0-01_覆盖窗口.md` 的窗口类就绪。
- 被依赖：全版本常驻基座；P6 设置面板复用 `Autostart` 开关。

## 风险 / 开放
- 管道心跳超时值需权衡（过短误杀、过长恢复慢），建议 3–5s。
- 双进程互相看门存在"双双卡死"边界，需 clean-shutdown 事件统一裁判。
- 注册表自启可能被安全软件拦，需引导用户手动放行 / 提供开始菜单快捷方式备选。
