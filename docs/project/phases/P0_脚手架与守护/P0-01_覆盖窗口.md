# P0-01 覆盖窗口（空壳）

> Phase：P0 · 脚手架 + 守护进程
> 上游：`v1开发计划.md` P0、`技术栈选型.md` §6（覆盖层核心）、`项目规格.md` §1/§4.2、`功能需求.md` FR-COV-01~05, 08~12
> 关联：`P0-02_守护进程.md`、`渲染架构设计.md` §3.1（输入拦截方案）

## 目标
双击 `overlay.exe` 后呈现一个铺满工作区的无边框全屏窗口：
- 盖住原生桌面图标与壁纸，**居普通应用窗口之下**（非置顶）；
- **不预留**任务栏条（任务栏由系统 Auto-hide 管理，其弹出时浮于覆盖层之上、可局部遮挡）；
- 窗口空白区**吸收输入**（不穿透到原生桌面）。

此时窗口内无内容（空壳），仅验证定位 / 层级 / 输入模型正确。

## 范围
**包含**
- WPF `Window` 无边框 + `AllowsTransparency=True`（支持后续透明度 / 毛玻璃）。
- P/Invoke 全屏矩形 = `SystemParameters.WorkArea`。
- 层级：`Topmost=False`，经 `SetWindowPos` 置于桌面图标之上、普通窗口之下。
- 输入拦截：默认整窗命中吸收；预留 `IsHitTestVisible` 选择性放行（方案 A）；备选方案 B 整窗 `WS_EX_TRANSPARENT` + `WM_NCHITTEST` 返回 `HTTRANSPARENT`（`技术栈选型.md` §6）。
- v1 锁定单屏 1920×1080（FR-COV-01），多屏留 v2（FR-COV-06/07）。

**不含**：原子 / 画布（P1）、背景 / 毛玻璃（P6）、任务栏接管（坚持系统 Auto-hide）。

## 关键设计
### 窗口样式
```xml
<Window WindowStyle="None" AllowsTransparency="True" Background="Transparent"
        ShowInTaskbar="False" Topmost="False" ResizeMode="NoResize"/>
```
- `ShowInTaskbar=False`：避免与 Auto-hide 触发冲突。
- 背景透明：绘制交给内部层 / 背景元素。

### 全屏定位
```csharp
var wa = SystemParameters.WorkArea;            // DPI 感知的工作区矩形
Left = wa.Left; Top = wa.Top; Width = wa.Width; Height = wa.Height;
```
Auto-hide 任务栏隐藏时 `WorkArea` 即全屏；任务栏弹出瞬间系统重排 `WorkArea`，窗口需监听 `WM_SETTINGCHANGE` 重贴。

### z 序（不置顶）
- 不调用 `HWND_TOPMOST`（会盖住任务栏触发区、普通窗口 / 设置）。
- 经 `SetWindowPos` 插入桌面 `WorkerW`/`Progman` 之上、普通窗口之下（z 基准枚举见 `渲染架构设计.md` §3）。

### 输入拦截（方案 A 优先）
- 窗口 `IsHitTestVisible=True` 命中即吸收，不向桌面穿透。
- 未来交互原子 `IsHitTestVisible=True` 收点击；展示原子 `False`（父窗口仍吸收，不穿透桌面）。
- 方案 B：`WS_EX_TRANSPARENT` 整窗穿透 + `WM_NCHITTEST` 选择性 `HTTRANSPARENT`，用于边界场景。

### P/Invoke 清单
| API | 用途 |
|-----|------|
| `SetWindowLong` / `GetWindowLong` | 改扩展样式（无边框、透明） |
| `SetWindowPos` | 设 z 序 / 位置 |
| `EnumWindows` / `FindWindow` | 定位桌面 Shell 窗口确定 z 基准 |
| `SystemParameters` (WPF) | 工作区矩形 |
| `RegisterHotKey` | 显隐快捷键（P0 预留接口） |

## 接口草图
```csharp
class OverlayWindow : Window {
    void ApplyFullscreenWorkArea();        // 贴 WorkArea + Auto-hide 重贴
    void SetZOrderBelowNormalWindows();    // z 序定位
    void EnableInputAbsorb(bool on);       // 方案 A 整窗吸收开关
}
static class NativeWindow {                // P/Invoke 封装
    void SetWindowLong(IntPtr hWnd, int idx, int style);
    void SetWindowPos(IntPtr hWnd, IntPtr insertAfter, int x,int y,int w,int h, uint flags);
}
```

## 技术选型
WPF 窗口 + Win32 P/Invoke（`技术栈选型.md` §6 覆盖层核心）。无 WebView2、无第三方窗口库。DPI `Per-Monitor v2`（`技术栈选型.md` §5.1），`WorkArea` 已缩放。

## FR 映射
FR-COV-01（单屏锁）、FR-COV-02（盖图标 / 壁纸）、FR-COV-03（居普通应用之下 / 非置顶）、FR-COV-04（输入拦截不穿透）、FR-COV-05（Auto-hide 不预留、弹出浮于其上）。

## 验收
双击 exe → 全屏覆盖层出现，原生桌面图标 / 壁纸被盖且点击无响应；普通应用窗口可正常覆盖覆盖层；任务栏可见可点，Auto-hide 弹出浮于其上；覆盖层不占任务栏按钮。

## 依赖与顺序
- 无前置（P0 起点）。
- 被依赖：P1 在本窗口内搭建渲染基座；`P0-02_守护进程.md` 负责拉起本窗口。

## 风险 / 开放
- z 序基准在不同 Windows 版本 / 壁纸引擎下差异，需实测 `WorkerW`/`Progman` 插入点。
- Auto-hide 重贴时机需可靠监听 `WM_SETTINGCHANGE`，避免被任务栏遮挡后不回退。
- 方案 A/B 取舍在 P1 原子交互明确后最终定（当前 P0 用方案 A 整窗吸收）。
