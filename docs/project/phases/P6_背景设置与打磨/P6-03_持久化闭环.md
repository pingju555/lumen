# P6-03 持久化闭环

> Phase：P6 · 背景 + 设置面板 + 持久化闭环 + 打磨
> 上游：`v1开发计划.md` P6、`功能需求.md` §3.0 / FR-SET-08、`技术栈选型.md` §6（持久化）+ §5.1（schema 迁移）
> 关联：`P2-06_持久化.md`（schema v1 升级自此）、`P3-04_预设库.md`（导入导出）、`P6-02_设置面板.md`

## 目标
将全量工程状态 —— **layers + behavior + widgets + atoms + pages + gv + 背景 + 设置** —— 聚合为 `Document` 序列化 JSON，完成"编辑 → 存盘 → 重启恢复 → 导入导出"闭环，并支持 schema 版本迁移（v1 → v1.1 → v1.2）。

## 范围
**包含**
- `Document` 聚合模型：screens[] → layers[]（含 atoms / gv / 背景）/ pages[] / widgets[] / behaviors[] / settings。
- 统一读写：启动时 `Load` 重建全部；编辑后 `Save`（增量或全量，v1 全量足够小）。
- 迁移：`Migrate(JObject)` 按 `version` 字段升到当前（v1.1 加 grid/widget/page，v1.2 加 behavior/file/workflow，破坏性仅 v2.0）。
- 导入 / 导出：整 `Document` 或单预设 / 单页面导出 JSON，便于备份分享。

**不含**：云同步（v2）；冲突合并（导入覆盖，v1 不 merge）。

## 关键设计
### Document 结构
```json
{
  "version": 2,
  "screens": [
    { "id":"DISPLAY1",
      "layers":[ { "kind":"Canvas","z":1,"opacity":1,"enabled":true,
                    "atoms":[...], "gv":[...], "background":{...} } ],
      "pages":[ { "name":"主","grid":40,"layers":[...] } ],
      "widgets":[...], "behaviors":[...], "settings":{...}
    }
  ]
}
```

### 读写 + 迁移
```csharp
class DocumentStore {
    int CurrentVersion = 2;
    Document Load() {
        var raw = JsonNode.Parse(File.ReadAllText(Path));
        var v = (int)raw["version"];
        var doc = v < CurrentVersion ? Migrate(raw, v) : Deserialize(raw);
        return doc ?? DefaultDocument();
    }
    void Save(Document d) => File.WriteAllText(Path, Serialize(d with version=CurrentVersion));
    Document Migrate(JObject raw, int from) { /* 逐版升级 */ }
    void Export(Document d, Stream out);
    Document Import(Stream json);
}
```

## 技术选型
`System.Text.Json`（`JsonNode` / `JObject`）+ `%LocalAppData%/Overlay/config.json`（`技术栈选型.md` §6 持久化 + §5.1 迁移）。零依赖。

## FR 映射
FR-SET-08（每屏×每层）、§3.0（结构）、导入导出（FR-SET / `P3-04`）；behavior/file/workflow 落盘（v1.2）。

## 验收
全状态编辑后存盘 → 重启完全一致；导出 JSON → 清空 → 导入恢复无丢失；旧版 v1 配置经 `Migrate` 升到 v2 不破。

## 依赖与顺序
- 依赖：`P2-06`(v1 雏形)、`P3-03`(页 DTO)、`P3-04`(预设/导入导出)、`P6-02`(设置写盘)。
- 被依赖：P6-04 打磨（NFR 抽检配置大小/速度）、全版本发布门禁（配置兼容）。

## 风险 / 开放
- 迁移测试：每版需样本旧配置回归，防 `Migrate` 漏字段（建议测试固化样例，见 `技术栈选型.md` §5 测试）。
- 导入覆盖风险：导入前提示备份（v1 可简单覆盖 + 自动备份上一版）。
- 大文档性能：v1 原子量小，全量序列化可接受；v2 多屏可能需分文件。
