# v2-01 多屏多实例

> 版本：v2.0 · 扩展与多屏（deferred 分支）
> 上游：`版本排期.md` §5、`v1开发计划.md` 备注（多屏多实例）、`功能需求.md` FR-COV-06/07、`技术栈选型.md` §5.2
> 关联：`P0-01_覆盖窗口.md`（单屏基座）、`P2-06_持久化.md`（`screens[]` 预留）、`P6-03_持久化闭环.md`

## 目标
将 v1 单屏单实例升级为**多屏**：单进程多 `Window`（每屏一个 `OverlayWindow`），每屏独立层栈 + 独立背景 + 独立页面，共享 gv / 配置。

## 范围
**包含**
- `ScreenManager`：`Screen.AllScreens` 枚举，为每屏创建 `OverlayWindow` + 独立 `LayerStack`（接 `P1-01`）+ 独立 `DirtyScheduler`（接 `P2-05`）。
- 每屏配置：`screens[{id, layers[], pages[], widgets[], ...}]`（`P2-06` 已预留 `screens[]`）。
- 共享 vs 独立：gv 全局共享；层 / 页面 / 原子每屏独立。
- 每屏 Auto-hide 任务栏独立处理（接 `P0-01`）。

**不含**：跨屏拖拽原子（v2.x 后续）；云同步多设备（v2.x）。

## 关键设计
```csharp
class ScreenManager {
    Dictionary<Screen, OverlayWindow> _wins = new();
    void StartAll() {
        foreach (var s in Screen.AllScreens)
            _wins[s] = new OverlayWindow(s) { DataContext = LoadScreen(s) };
    }
}
```
- 各屏 `OverlayWindow` 贴各自 `Bounds`（`SystemParameters` 无多屏工作区直接 API，需用 `Screen.Bounds` 减任务栏或 `MONITORINFO`）。
- 单进程多 Window 共享一个 `Dispatcher` 线程（WPF 单线程模型），或每屏独立线程（复杂）；v1 单 Dispatcher 足够。

## 技术选型
`System.Windows.Forms.Screen.AllScreens`（或 `System.Windows.SystemParameters` 多屏）、`MONITORINFO` P/Invoke（`技术栈选型.md` §5.2 多屏行）。无第三方。

## FR 映射
FR-COV-06（多屏覆盖）、FR-COV-07（每屏持久化）。

## 验收
多显示器的每台屏均出现独立覆盖层；各屏层 / 页面独立配置；gv 跨屏共享。

## 依赖与顺序
- 依赖：P0/P1/P2/P3/P6 全量稳定 + `P2-06` screens 结构。
- 被依赖：v2 其余 deferred（天气 / 网页 / LLM 可在指定屏渲染）。

## 风险 / 开放
- **单进程多 Window vs 多进程**：`版本排期.md` §8 待拍板。单进程共享 Dispatcher 简单但一屏崩全崩；多进程隔离但 IPC 复杂。
- 多屏 DPI 各异：`Per-Monitor v2` 下每窗独立 DPI，需逐窗感知（`P1-02` 坐标需绑屏）。
- Auto-hide 多屏任务栏独立触发，重贴逻辑需逐屏。
