# v2-02 天气

> 版本：v2.0 · 扩展与多屏（deferred 分支）
> 上游：`版本排期.md` §5、`功能需求.md` §6.5 / FR-CW-06~09
> 关联：`P4-04_公式Provider闭环.md`（IDataProvider 路由）、`P6-03_持久化闭环.md`（Key 配置）

## 目标
接入天气数据（外部 API），经 `cw()` 类公式 / 部件显示当前天气、温度、预报。需数据源 + API Key（**待用户拍板**，见 `版本排期.md` §8）。

## 范围
**包含**
- `WeatherProvider : IDataProvider`：HTTP 拉取天气 JSON → 暴露 `cw(temp)` / `cw(desc)` / `cw(city)` 等。
- 缓存 + 定时刷新（如 10–30min，省配额）。
- 配置：数据源选择 + API Key（存 `P6-03` 配置，注意加密 / 不落明文风险）。

**不含**：天气动画背景（可结合 v2-05 动态壁纸）；多日预报 UI 复杂（v1 仅当前）。

## 关键设计
```csharp
class WeatherProvider : IDataProvider {
    HttpClient _http = new();
    string _key;                              // 用户配置
    async Task Refresh() => _cache = await _http.GetFromJsonAsync($"https://api.x/now?key={_key}&city={_city}");
    public Value Get(string name, Value[] a) => ReadFrom(_cache, name);
}
// 注册进 P4-04 CompositeDataProvider（路由 "cw"）
```

## 技术选型
`System.Net.Http.HttpClient`（原生，`技术栈选型.md` §5 deferred 分支行）。零 NuGet。

## FR 映射
FR-CW-06~09（天气数据 / 显示 / 配置）。

## 验收
配置 Key + 城市 → `$cw(temp)$` 显示实时温度；定时刷新；断网回退上次 / 静态。

## 依赖与顺序
- 依赖：`P4-04`(Provider 路由)、`P6-03`(Key 配置)。
- 被依赖：v2 天气部件 UI（基于六类原子）。

## 风险 / 开放
- **数据源与 Key 未定**：需用户选供应商（OpenWeather / 和风等）并填 Key；密钥安全（避免明文配置 / 内存残留）。
- **外部 API 违反"全本地零外部"**：v1 原则为本地，天气是首个外部依赖，需用户明确拍板（开放决策 §8）。
- 配额 / 限流 / 隐私（位置）。
