# v2-05 动态壁纸 / 视频背景

> 版本：v2.0 · 扩展与多屏（deferred 分支）
> 上游：`版本排期.md` §5、`功能需求.md` FR-BG-06
> 关联：`P6-01_背景渲染.md`（背景体系扩展）

## 目标
支持动态壁纸 / 视频背景：将视频（或动态图像）作为壁纸层背景，而非静态图片。v1 明确**不做**，v2 评估引入。

## 范围
**包含**
- 视频背景：`MediaElement`（WPF）或 `WebView2`（v2-03）承载视频循环播放。
- 动态图像：GIF / APNG / 着色器动画背景。
- 性能控制：暂停不可见屏视频（接 `P2-05`）、降码率守 NFR。

**不含**：音频（视频壁纸默认静音）；交互式动态壁纸（v1 基础）。

## 关键设计
```csharp
class VideoBackground : Background {
    MediaElement _vid = new() { LoadedBehavior = Play, IsMuted = true, Loop = true };
    UIElement Render() => _vid;   // 挂 WallpaperLayer 最底
}
```
- 静音播放：避免与媒体 `mi()`（P4-02）冲突（壁纸视频静音不进 SMTC）。
- 多屏（v2-01）：每屏独立视频，不可见屏 `Pause` 省资源。

## 技术选型
WPF `MediaElement`（原生）或 `WebView2`（`v2-03`）。无第三方。

## FR 映射
FR-BG-06（动态壁纸 / 视频）。

## 验收
壁纸层播放静音循环视频；切到不可见屏时暂停；性能不破 NFR。

## 依赖与顺序
- 依赖：`P6-01`(背景接口)、`P2-05`(降频暂停)、`v2-01`(多屏)。
- 被依赖：v2 动态主题。

## 风险 / 开放
- **性能 / 体积**：视频解码占 CPU/GPU，可能破 NFR-02（空闲 CPU~0）——需仅在可见屏播放且空闲时仍可降频。
- 格式支持：`MediaElement` 依赖系统解码器（如 HEVC 需授权）；跨机兼容需评估。
- 与毛玻璃（P6-01）叠加的视频模糊开销。
