# 公式引擎设计（Formula Engine）

> 依据：`klwp_aosp_release_382.apk` 逆向（dex 类名单 + `assets/komponents/*.komp` 实测）+ Kustom 官方文档交叉验证。
> 决策：**v1 即包含公式求值引擎**（用户拍板）。属性三元组中"公式"分支自 v1 起生效。
> 目标：设计桌面端原生公式引擎，复用 KLWP 的 `$...$` 语法与函数范式，数据源改为 Windows 原生（Win32/PDH/WMI/SMTC）。
> **完整函数全集（51 个，dex 代码确证 + 预设实测）见 [`公式函数参考.md`](./公式函数参考.md)，本文 §2 为其子集，以该文为准。**

---

## 1. 引擎架构（从 KLWP 提炼）

| 要素 | KLWP 实现 | 我们的实现 |
|------|-----------|-----------|
| 公式定界 | `$表达式$`，可嵌套 | 同 `$...$`，可嵌套 |
| 求值时机 | 属性绑定公式，按刷新 tick 重算 | 同：原子属性绑定公式，由数据变更/定时触发重算 |
| 函数命名空间 | 2 字母前缀（`df`/`si`/`gv`…） | 同前缀，便于用户迁移直觉 |
| 变量 | `gv(name)` 读全局变量 | 同；全局变量跨层共享 |
| 类型 | 字符串/数字/颜色/布尔（隐式转换） | 显式 `Value` 枚举（见 §5） |
| 数据源 | 全本地（系统/电池/音乐/天气/位置） | 全本地；天气/位置 v1 暂不接（预留函数名） |

**核心范式不变**：一个属性三选一 —— 静态值 / 全局变量 / 公式。公式是最强形态，可读取系统状态并响应变化。

## 3. 运算符与文法

```
formula      := expr
expr         := if_expr | or_expr
if_expr      := 'if' '(' cond ',' expr [',' expr] ')'
cond         := expr cmp expr            # = > >= < <= != ~=
or_expr      := and_expr ('|' and_expr)*
and_expr     := add_expr ('&' add_expr)*
add_expr     := mul_expr (('+'|'-') mul_expr)*
mul_expr     := unary (('*'|'/') unary)*
unary        := '-' unary | primary
primary      := NUMBER | STRING | COLOR | IDENT '(' [args] ')' | IDENT | '(' expr ')'
args         := expr (',' expr)*
STRING       := '"..."' | "'...'"
COLOR        := '#RRGGBB' | '#AARRGGBB'
IDENT        := [a-z]{2}                 # 2 字母函数/变量名
```

- 字符串拼接用 `+`（如 `if(bi(level)=100,"满电","剩 "+bi(level)+"%")`）。
- `~=` 为正则匹配（`df(MMMM) ~= "a"`）。
- 函数可嵌套、可出现在任意参数位。

## 4. v1 函数子集（落地范围）

必做（覆盖 v1 原子：时钟/日历、系统监控、启动坞、媒体）：

```
df  tf  si  bi  gv  if  tc  mi  mu
```

- `df`/`tf` → 时钟/日历原子。
- `si`(扩展桌面字段) → 系统监控原子（CPU/内存/网速/磁盘/进程）。
- `bi` → 电量（并入系统监控或独立）。
- `gv` → 全局变量读写（主题色/用户配置）。
- `if`/`tc` → 条件显隐、文本截断（便签/媒体标题）。
- `mi`/`mu` → 媒体控制原子（SMTC）。

预留（函数名存在，求值器 stub）：
```
wi  wf  li  cc  ca  ci  nc  ns  sh  fl  bp  br  ai
```

## 5. C# 求值器架构

```
┌──────────────────────────────────────────────┐
│  Atom 属性求值入口                              │
│  prop = { static?, global?, formula? }         │
│  → formula 优先 → gv(global) → static          │
└───────────────┬──────────────────────────────┘
                │ 含 $...$ 则进入
┌───────────────▼──────────────────────────────┐
│  Engine.evaluate(formula, ctx)                 │
│   1. Lexer    : 拆分字面量 + $公式$ 段          │
│   2. Parser   : 递归下降 → AST                  │
│   3. Evaluator: 遍历 AST                        │
│        - 字面量 → Value                          │
│        - 二元运算 → 求值左右 + 应用算子          │
│        - 函数调用 → FunctionRegistry.dispatch   │
└───────────────┬──────────────────────────────┘
                │
┌───────────────▼──────────────────────────────┐
│  FunctionRegistry（2 字母 → Fn(args, ctx)）     │
│   df tf si bi gv if tc mi mu ...               │
│   每个函数委托给对应 DataProvider                │
└───────────────┬──────────────────────────────┘
                │
┌───────────────▼──────────────────────────────┐
│  DataProvider（trait，懒加载 + 缓存 + tick 刷新）│
│   ClockProvider   → 系统时间                    │
│   SystemProvider  → PDH/WMI（CPU/内存/网速/磁盘）│
│   BatteryProvider → GetSystemPowerStatus        │
│   MusicProvider   → SMTC（标题/状态/控制）       │
│   GlobalsProvider → 读全局变量表                 │
└──────────────────────────────────────────────┘
```

**Value 类型**：
```csharp
enum ValueType { Num, Str, Color, Bool, Null }
class Value {
    ValueType Type;
    double Num;     // f64
    string Str;
    uint   Color;   // #AARRGGBB
    bool   Bool;
}
```
- 隐式转换：数字↔字符串；颜色↔字符串(`#RRGGBB`)；条件运算要求 Bool。
- 错误策略：函数未命中/类型错 → 返回 `Str("")` 或 `Null`，不崩溃（保持桌面稳定）。

**刷新模型**：
- 时钟类：1s tick 重算 `df/tf` 相关原子。
- 系统类：`SystemProvider` 内部 1s 轮询 PDH，脏标记触发依赖原子重算。
- 音乐类：订阅 SMTC 事件回调，状态变即重算 `mi` 相关。
- 全局变量：用户改 `gv` → 标记依赖该变量的原子重算。

## 6. 与属性三元组 / 多层架构衔接

- 每层、每个原子、每个属性都可绑定公式 → 公式作用域**全局**（跨层共享 `gv` 与 `si`）。
- 公式结果驱动：文本原子内容、形状尺寸/颜色、进度原子百分比、图标选择（`if(wi(code)=CLEAR, Sun, Moon)` 范式）、容器显隐。
- 持久化：公式字符串随属性三元组存入 JSON（见 `KLWP 逆向分析.md` §8.6 schema）。
- 编辑态：属性面板提供"公式"输入 + 函数自动补全 + 实时预览（§1.7 FR-LAYER-07）。

## 7. 风险与对策

| 风险 | 对策 |
|------|------|
| 公式死循环（如 `gv` 自引用） | 求值深度上限 + 依赖图环检测 |
| 函数名与未来扩展冲突 | 2 字母命名空间预留注册表，新增需显式登记 |
| Shell(`sh`) 安全 | v1 不含；v1.x 加白名单/沙箱 |
| 性能（大量原子 + 高频 tick） | 脏标记增量重算，非全量；1s 节流 |
| 类型错误导致渲染异常 | Value 容错转换，错误回退静态值 |

---

## 8. 结论

公式引擎是原子化桌面的"灵魂"，v1 即落地。已确认 KLWP 词表（`df/tf/si/bi/gv/if/tc/mi/mu` 等）可直接映射到 Windows 原生数据源，C# 求值器采用 Lexer→Parser→AST→Registry→Provider 五段式，配合属性三元组与全局变量，实现"属性可计算、可响应状态"。天气/位置等外部源 v1 仅占位，后续接入。
