# Lumen 编辑器重构 — 界面原型设计（基于 KLWP 参考）

> 文档目的：分析 20 张 KLWP（Kustom Live Wallpaper）编辑器截图，提取其界面原型逻辑，映射到 Lumen 现状，形成编辑器及相关界面体系重构的设计基线。
> 来源：用户截图 20 张（编辑器属性面板 / 项目列表 / 添加组件 / 公式编辑器 / 流程创建菜单）
> 关联代码：`src/lumen/Ui/PropertyEditorPanel.xaml.cs`、`LumenWindow.xaml.cs`、`Ui/FormulaTextBox`、`Ui/TreeWindow`、`Atoms/*`、`Actions/Flow.cs`
> 日期：2026-07-18

---

## 0. 分析结论速览

| 维度 | KLWP 现状（参考） | Lumen 现状 | 差距 |
|------|------------------|-----------|------|
| 整体布局 | 三栏常驻（列表/属性/工具）+ 按需弹窗 | 独立 TreeWindow + PropertyEditorPanel + 散落工具 | TreeWindow 未内嵌 |
| 属性分组 | 6 Tab（形状/油漆/图层/位置/动画/触摸） | 扁平列表 | 需改为 Tab 分组 |
| 属性行控件 | 步进器◀▶ + 微调−+ + 滑块 + 下拉 + □公式 | 文本框为主 | 控件风格待统一 |
| 公式编辑器 | 实时预览 + 输入 + 工具栏 + 函数分类网格(2页) | 输入 + 工具栏 | 缺预览区 + 分类网格 |
| 添加组件 | 11 卡片网格（汉堡菜单触发） | 右键/菜单新建 | 缺卡片网格面板 |
| 图层能力 | 混合模式/滤镜/模糊/纹理 | ImageAtom 有 Source+Opacity | 混合/滤镜/模糊缺失 |

**已对齐（今天落地）**：动画条件 `animWhen` + 动作参数下拉化；触发器→流程(Flow) 重命名 + RunFlow/SetVar/Delay/ReadFile 四种动作。

**Lumen 差异化优势（保留）**：Shape 14 种程序化质感（Frosted/Glass/Metal/Neon/...）是 KLWP 没有的，重构时不应删除。

## 1. 全局布局架构

KLWP 编辑器是「**三栏常驻 + 按需弹窗**」结构：

```
┌─────────────────────────────────────────────────────────────┐
│ Screen（全屏编辑画布）                                          │
├──────────────┬────────────────────────┬──────────────────────┤
│ 左栏          │ 中栏                    │ 右栏                  │
│ ProjectList  │ PropertyEditor         │ 工具栏                │
│ (项目列表)    │ (属性编辑器 6 Tab)      │ ☰ 汉堡(添加组件)       │
│ 背景/层/位置/ │ 形状/油漆/图层/位置/     │ 💾 保存               │
│ GV/流程/快捷  │ 动画/触摸              │ 🔄 刷新               │
│              │                        │ ＋ 添加               │
└──────────────┴────────────────────────┴──────────────────────┘
        │                  │                      │
        └──── Overlay 弹窗层（按需） ────────────┘
         · 添加组件面板 (AddComponent)
         · 公式编辑器 (FormulaTextBox)
         · 流程创建菜单 (FlowBuilder)
```

| 栏位 | 职责 | Lumen 现状对照 |
|------|------|---------------|
| 左栏 ProjectList | 全页面原子树；Tab 切换背景/层/位置/全局变量/流程/快捷方式 | `TreeWindow` 为独立窗口，未内嵌主窗 |
| 中栏 PropertyEditor | 选中项的属性，按语义 6 Tab 分组 | `PropertyEditorPanel`（已实现，但为扁平列表，无 Tab 分组） |
| 右栏 工具栏 | 保存/刷新/添加(+)/汉堡菜单 | 功能散落，无统一工具栏 |
| 弹窗层 Overlay | 公式编辑器/添加组件/流程创建 | `FormulaTextBox` 已有；其余缺 |

**重构方向**：将 TreeWindow 内嵌为左栏，属性面板改为中栏 6 Tab，新增右栏工具栏 + Overlay 弹窗体系。

## 2. 属性编辑器 — 6 Tab 体系

### 2.1 通用属性行结构（所有 Tab 复用）

每一行统一格式：

```
[icon] 标签名         [控件区]        [□公式按钮]
                          ↑
        ┌────────────┬────────────┬────────────┐
        │ 步进器 ◀▶   │ 微调 − +    │ 滑块 ─●─    │ (按字段类型选)
        │ 下拉 ▾      │ 文本框      │            │
        └────────────┴────────────┴────────────┘
```

- **□ 公式按钮**：空心 = 未绑定公式；实心红 ■ = 已绑定公式；点击 → 打开 FormulaTextBox。
- **数值控件规范**：
  - `◀▶ 步进器`：大步长（如 ±10），快速粗调
  - `− + 微调`：小步长（如 ±1），精确微调
  - `滑块`：连续值（圆角/模糊/暗淡/速度/角度）
  - `下拉`：枚举（形状类型/风格/混合模式/锚点/动作类型）

### 2.2 Tab 详细字段对照

#### Tab 1 — 形状 Shape（✅ 基本对齐）

| 字段 | KLWP 控件 | Lumen 现状 |
|------|----------|-----------|
| 形状类型 | 下拉（矩形/椭圆/...） | ✅ `Type` 枚举下拉 |
| 宽度 | 步进器 + 微调 + □ | ✅ `Width`（当前纯文本框） |
| 高度 | 步进器 + 微调 + □ | ✅ `Height` |
| 圆角半径 | 滑块 | ✅ `Radius` |
| 旋转角度 | 下拉/数值 + □ | ✅ `Rotation` |

**重构**：宽度/高度改用步进器+微调控件，每行加 □公式按钮。

#### Tab 2 — 油漆 Paint（⚠ Lumen 有差异化优势）

| 字段 | KLWP 控件 |
|------|----------|
| 风格 | 下拉（填充/描边） |
| 颜色 | #AARRGGBB 文本 + 色块预览 + 取色器 |

**Lumen 映射**：Lumen 有 14 种程序化质感（Frosted/Glass/Plastic/Metal/Neon/Matte/Wood/Marble/Carbon/Holographic/Paper/Fabric/Liquid）。重构方案：
- 「风格」下拉扩展为三级：**纯色填充 / 描边 / 质感**
- 选「质感」时展开质感下拉（14 种）+ 颜色（基调）
- 这是 KLWP 没有的差异化能力，保留并强化

#### Tab 3 — 图层 Layer（⚠ 增强候选，最大差距）

| 字段 | KLWP 控件 | Lumen 状态 |
|------|----------|-----------|
| 纹理/位图源 | 选择位图 + 🔍浏览 + ✂裁剪 | ImageAtom 有 Source |
| 混合模式 | 下拉（居中=Normal） | ❌ 缺失 |
| 滤镜 | 下拉（无） | ❌ 缺失 |
| 模糊 | 滑块 | ❌ 缺失（Shape 用质感模拟部分） |
| 暗淡/透明度 | 滑块 | ✅ `Opacity` |

**重构候选**：混合模式（XOR/Difference/Saturation 等）+ 滤镜 + 模糊 = v1.2 渲染管线增强。Shape 质感可部分替代，但逐形状 Mask/Blurred 是独立方向。

#### Tab 4 — 位置 Position（✅ 已有，Lumen 更优）

| 字段 | KLWP 控件 | Lumen 对应 |
|------|----------|-----------|
| 锚点 Anchor | 下拉枚举（中心左/上中/...） | ✅ 九宫格 GridModel（点击选，更直观） |
| X 偏移 | 步进器 + 微调 + □ | ✅ XY 偏移 |
| Y 偏移 | 步进器 + 微调 + □ | ✅ |

**重构**：控件风格统一为步进器+微调；九宫格锚点保留（优于 KLWP 文字下拉）。

#### Tab 5 — 动画 Animation（✅ 今日落地，动作集方向差异）

| 字段 | KLWP 控件 | Lumen 状态 |
|------|----------|-----------|
| 触发条件 animWhen | 公式输入 + □ | ✅ 今天落地（`AnimWhenProp`） |
| 动作类型 | 下拉（滚动/缩放/旋转/平移/页面切换） | ⚠ 不同（Zoom/Bounce/Pulse/Fade） |
| 规则 | 下拉（中心/边缘/...） | ⚠ 部分 |
| 中心点/原点 | 下拉（屏幕1/自身中心/...） | ⚠ 缺 |
| 速度 | 滑块 | ✅ 有参数 |
| 角度 | 滑块 | ✅ 有参数 |

**关键差异**：KLWP 动画动作与触摸联动（交互响应型）；Lumen 动画偏视觉装饰。两者可互补——未来 Lumen 可补「滚动/平移」类交互动画。

#### Tab 6 — 触摸/流程 Touch & Flow（✅ 今日落地）

- **空状态**：提示「该模块没有触摸事件，点击右上角 + 添加」
- **+ 弹出菜单**：延迟 / 设置全局变量 / 设置局部变量 / 网络信息 / 公式 / ...
- **Lumen 今日已落地**：`AtomTrigger`→`Flow` 重命名；新动作 `RunFlow`/`SetVar`/`Delay`/`ReadFile`（全异步 + 循环引用防护）

### 2.3 属性编辑器重构要点

1. 从扁平 `StackPanel` 列表改为 **6 Tab 分组**（`TabControl` 或自绘 Tab 栏）
2. 所有属性行统一为「icon + label + 控件 + □」四段式
3. 数值字段按类型自动选控件（步进器/微调/滑块/下拉）
4. □ 按钮全局接入 FormulaTextBox，绑定状态可视化

## 3. Overlay 弹窗面板

### 3.1 添加组件面板 AddComponent（截图17）

**触发**：右栏汉堡菜单或 `+` 按钮
**形式**：11 卡片网格，每卡 = 图标 + 名称 + 两行描述

| 卡片 | 描述摘要 | Lumen 映射 |
|------|---------|-----------|
| 组件 | 预建模板（信号指示/电池图标...） | ❌ 缺（Templates） |
| 文本 | 任意文本，可选例子 | ✅ TextAtom |
| 形状 | 矩形/圆/颜色渐变填充 | ✅ ShapeAtom（Rect/Ellipse） |
| 图像 | 选择位图 | ✅ ImageAtom |
| 图标 | 可缩放矢量图标 | ✅ IconAtom |
| 进度 | 进度条（风格/形状/颜色） | ✅ ProgressAtom |
| 序列 | 时间/日期/电池序列 | ❌ 缺（Sequence） |
| 文本变形 | 单行文本变形风格 | ❌ 缺（TextMorph） |
| 堆叠组 | 横向/纵向排列容器 | ✅ StackGroupAtom |
| 重叠组 | 允许重叠的分组容器 | ✅ OverlapGroupAtom |
| 动态图片 | GIF 动图 | ❌ 缺（Gif） |

### 3.2 公式编辑器 FormulaTextBox（截图18-20）

**三层结构**：

```
┌─────────────────────────────────────┐
│ [文本预览区]  14:43:52               │  ← 实时解析结果
├─────────────────────────────────────┤
│ [输入区]  $df(hh:mm:ss)$ |          │  ← monospace 公式文本
├─────────────────────────────────────┤
│ 🌐 🎨 ★  B I T U Z                  │  ← 工具栏
├─────────────────────────────────────┤
│ 函数分类网格（2 页）：                │
│  ★收藏公式  📅日期  📍位置  ☁️天气    │
│  🔋电池   ⏱间隔  📊状态栏 🎵音乐     │
│  🌐网络  💻系统  ➗数学  🎨颜色提取    │
│  🔧颜色生成  ✂日期解析 GV 广播       │
│  📜GlobalHistory 命令  ⏲计时        │
│  🔁For循环  ⊞局部变量  🌐网站帮助     │
└─────────────────────────────────────┘
```

- **实时预览区**：显示公式求值结果（如 `14:43:52`），输入即更新
- **工具栏**：🌐全局变量 / 🎨调色板 / ★收藏 / B I T U Z（文本样式）
- **函数分类网格**：点击追加、长按替换；按领域分组（2 页）

**Lumen 差距**：现有 FormulaTextBox 有输入+工具栏，但**缺实时预览区 + 函数分类网格**。这是用户体验最大提升点。

**重构方案**：
- 加 `PreviewArea`（绑定 `FormulaEngine.Eval` 实时刷新）
- 加 `FunctionCatalogGrid`（数据来自 `Formula/FunctionCatalog.cs`，按 `Category` 分组）
- 分类可复用 `FunctionCatalog` 的 `func.*` 语言包 key

### 3.3 流程创建菜单 FlowBuilder（截图14）

`+` 弹出动作类型菜单：延迟 / 设置全局变量 / 设置局部变量 / 网络信息 / 公式 / ...
Lumen 已通过 `AtomAction` 枚举 + `BuildCompositeArg` 实现，菜单项可直接映射。

## 4. 项目列表 ProjectList（截图9-16）

**结构**：标题栏（项目 N/total + 搜索🔍）+ Tab 栏 + 列表

**Tab 栏**：背景 / 层 / 位置 / 全局变量 / 流程 / 快捷方式

| Tab | 内容（截图对应） |
|-----|----------------|
| 背景 | 类型（纯色）+ 颜色（#FF333333） |
| 层 | 可见性锁 / 比例 / 位置(GPS默认) / 时区 |
| 位置 | 四边距（左/右/上/下各 0） |
| 全局变量 | 位图选择器（背景壁纸） |
| 流程 | Flow000（禁用状态示例） |
| 快捷方式 | 空状态提示 |

**列表项结构**：
```
[类型色点] 名称 · 子类型           [□]
           参数摘要行（展开显示）
```
- 类型色点：按原子类型着色（形状/重叠组/文本...）
- 参数摘要：如「矩形, 圆角=#FF000000, 长度=284.0, 高度=4.0...」
- 右侧 □：公式入口

**Lumen 映射**：`TreeWindow` 当前为独立窗口，列表项有类型图标但无参数摘要、无 Tab 分组。重构为内嵌左栏 + Tab 切换 + 摘要展开。

## 5. UI 设计模式提炼（重构规范）

从截图提取的通用模式，作为 Lumen 重构的 UI 规范：

| 模式 | KLWP 做法 | Lumen 重构目标 |
|------|----------|---------------|
| **属性行** | icon + label + 控件(stepper/slider/dropdown) + □公式按钮 | 所有属性行统一四段式；□=FormulaTextBox 入口 |
| **数值控件** | ◀▶ 步进(大步长) + −/+ 微调(小步长)；滑块=连续值；下拉=枚举 | 替代纯 TextBox，按字段类型选控件 |
| **公式绑定视觉** | 空心□=未绑；实心红■=已绑 | 统一视觉语言，一眼识别动态值 |
| **列表项** | 类型色点 + 名称·子类型 + 参数摘要(展开) | TreeWindow 改内嵌左栏，支持摘要 |
| **空状态** | 文案引导 + 右上角 + 创建入口 | 所有空 Tab/空列表统一此模式 |
| **Tab 分组** | 属性按语义分组(形状/油漆/图层/位置/动画/触摸) | PropertyEditor 改 6 Tab |
| **弹窗面板** | Overlay 半屏/全屏，不阻塞主画布 | FormulaTextBox 增强；新增 AddComponent |

## 7. 重构优先级与路线图

| 优先级 | 改动 | 收益 | 工作量 |
|--------|------|------|--------|
| **P0** | PropertyEditor 改为 6 Tab 分组（对齐 KLWP） | 用户心智模型统一；属性不再一坨列表 | 中（重排 BuildCommonProps） |
| **P0** | 属性行控件统一化（步进器/滑块/下拉/□公式） | 操作效率大幅提升；公式绑定一目了然 | 大（新控件 + 全属性改写） |
| **P1** | FormulaTextBox 加实时预览区 + 函数分类网格 | 降低公式学习成本；发现率提升 | 中（UI 重构，引擎不动） |
| **P1** | TreeWindow 内嵌主窗口左栏（替代独立窗口） | 三栏一体，不用来回切窗口 | 大（窗口结构改动） |
| **P2** | 新增 AddComponent 卡片网格面板（汉堡触发） | 新建原子更直观 | 中 |
| **P2** | 图层混合模式 + 滤镜 + 模糊 | 功能补齐 KLWP 级别 | 大（渲染管线改动） |
| **P3** | 序列/文本变形/GIF/预建组件原子 | 组件库完整度 | 各自独立 |
