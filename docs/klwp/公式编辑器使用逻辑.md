# KLWP / KWGT 公式编辑器使用逻辑（逆向）

> 来源：`kwgt_aosp_release_382/` 解包产物（19 个 `classes*.dex`，Kustom v382）。本文聚焦现有逆向文档**未覆盖的「公式编辑器本身的使用逻辑 / UX 细节」**，与 `KLWP 逆向分析.md`（模型）、`Kustom 原子模型.md`（原子）、`公式函数参考.md`（函数集）互补。
> KLWP 与 KWGT **共享同一渲染/编辑器引擎**，本文以 KWGT 抽取为准，未重复抽 KLWP。
> 方法：Python 解析 dex 字符串表（`string_ids` → `string_data` MUTF-8），筛 `kode/Kustom/Kode/komponent` 取得 Kustom 作用域串，再按编辑器/公式/全局变量/guide 关键词提取类描述符与 UI 文本。

---

## 0. 一句话结论

Kustom 的公式编辑器**不是单一大输入框**，而是一套
**「属性级唤起 → 注解高亮输入 → 函数目录插入 → 全局变量选择 → 实时预览/校验」**
的 Compose 编辑器框架。每个可计算属性都从「三态开关」进入公式模式，打开一个
`FormulaTextField`，配合函数速查、全局变量对话框与实时预览完成编辑。

> 这正是现有 md 缺的那一层：之前只写了「模型（属性三元组）/ 函数集 / 变量表」，
> 没写「编辑器这个东西到底怎么用、长什么样」。

---

## 1. 编辑器如何被唤起（使用入口）

- 每个原子属性有三态（静态 / 全局 / 公式，见 `Kustom 原子模型.md §4`）。切到「公式」即唤起编辑器。
- 底层由意图 **`org.kustom.intent.action.EDIT_KODE`** 驱动；公式语言官方名叫 **"Kode"**（权威键 `org.kustom.KODE` / `org.kustom.extra.KODE`）。
- 存在一个**可列举的「Kode Editors」集合**（`Unable to list Kode Editors` 这条错误串证明）——即不同属性类型（文本 / 颜色 / 数字 …）可挂载不同的 Kode 编辑器实例。
- 编辑器宿主 Activity：`org.kustom.lib.editor.EditorActivity`（含 `EditorDrawerHeader` 导航抽屉、`BasePrefFragment` 设置页）。

|------|-----------|------|

> **硬编码的 guide / 错误 / 类名串**。

## 3. 编辑器的实际使用流程（UX）

1. 在属性面板点该属性的「公式」入口 → 打开 `FormulaTextField`。
2. 输入 `$...$` 包裹的表达式；输入时 **`AnnotatedFormula` 实时语法高亮**（函数名 / 分隔符 / 字符串异色）。
3. **插入函数**：经「函数速查 / 目录」（编辑器内置 `## Formula Quick Reference`，完整目录见 `GET /api/formulas`）点选，插入**带占位参数的模板**（如 `df(,)`）。
4. **插入全局变量**：经 `PreviewGlobalDialog` 全局变量对话框选择 `gv(name)`，支持 `gv(name, default, index)`。
5. **实时预览**：编辑器内联 `PresetPreview` / `PreviewRenderFlowTask` 显示公式求值结果（文本 / 颜色 / 尺寸即时反映）。
6. **校验与报错**：非法公式通过 `InputTextFieldError` 状态 + `ErrorSnackBar` / `ErrorMessage` 即时反馈；批量操作返回结构化错误 `{"error":{"code":...,"message":...}}`。
7. **颜色属性同样支持公式**：如 `paint_color = $if(si(darkmode)=0, #FF1B1B1D, #FFFFFFFF)$`；编辑器对颜色公式给出暗色模式指引。

## 4. 全局变量（gv）编辑器逻辑

- **类型比之前文档更全**：`COLOR / NUMBER / TEXT / FONT / LIST / SWITCH`（较 `Kustom 原子模型.md §5` 的清单**多出 `LIST`**）。
- 创建：`addGlobal` 动作（`key` / `type` / `value`），`value` 自身可为公式。
- 绑定：`bindGlobal` 动作将属性与全局变量绑定；或属性直接引用 `gv(name)`。
- 作用域：**preset 级（主题化）**，跨属性 / 层共享。
- 编辑器内通过 `PreviewGlobalDialog` 选择 / 插入。

## 5. 内置「Kustom Editor Guide」文档结构（编辑器内帮助）

解包字符串证实编辑器内置一份使用指南，章节如下——它本身就是公式编辑器对用户的
**「使用逻辑说明书」**：

```
# Kustom Editor Guide
## Formula Quick Reference      （公式语法速查）
## Globals (Theming)            （全局变量 / 主题化）
## Properties Quick Reference   （属性速查）
### Common Formulas             （常见公式）
### Dark Mode Colors            （暗色模式配色）
## Error Codes                  （错误码）
```

语法要点（来自 guide 串）：

- `Kustom formulas use $fn(args)$ syntax. Text outside delimiters is literal.`
- 参数逗号分隔；字符串除非含逗号 / 括号否则不需引号。
- 嵌套：`$tc(upper, df(EEEE))$`；条件：`$if(bi(level)<20, LOW, OK)$`。

## 6. 常见公式范式（来自 guide / 预设实测）

| 类别 | 范式 |
|------|------|
| 日期时间 | `` `$df(EEEE, MMMM d)$` `` 、 `` `$df(hh:mm)$` `` |
| 条件 | `` `$if(bi(level)<20, LOW, OK)$` `` |
| 文本处理 | `` `$tc(upper, df(EEEE))$` `` |
| 颜色条件 | `` `$if(bi(level)<20, #FFFF0000, #FF00FF00)$` `` |
| 调色 | `` `$ce(#FF0000, alpha, 50)$` `` 、 `` `$ce(#FF0000, lum, 45)$` `` |
| 调色板取色 | `` `$bp(vibrant, mi(cover))$` `` （从专辑封面取色） |
| 系统强调色 | `` `$si(systemaccent)$` `` （Material You） |
| 响应式 | 全局 `side` 然后 `` `shape_width = $gv(side)/4$` `` |
| 暗色 | `` `$if(si(darkmode)=0, #FF1B1B1D, #FFFFFFFF)$` `` |

## 7. 编辑器 API / 批量操作模型（程序化编辑）

解包字符串含一份「Kustom Editor API」文档与 MCP 入口（`McpEditorProvider`），揭示
公式 / 全局变量是如何被**程序化设置**的——其动作模型与三态绑定完全一致：

- 端点：`GET /api/preset/tree`、`GET /api/preset/module-types`、`GET /api/preset/modules/{id}/properties`、`POST /api/preset/batch`、`GET /api/formulas`、`GET /api/preset/render`。
- 批量动作（`action`）：`addModule`、`addGlobal`、`setProperty`、`setFormula`、`bindGlobal`。
- 默认 `stopOnError=false`：批量遇错继续，一次性返回所有错误。
- 错误码：`INVALID_PROPERTY_VALUE (400)`、`UNKNOWN_METHOD (404)`、`NO_PRESET (503)`、`MODULE_NOT_FOUND (404)`、`MISSING_PARAM (400)`、`INVALID_PROPERTY_KEY (400)`。
- 示例：
  ```json
  {"action":"addGlobal","params":{"key":"bg","type":"COLOR",
   "value":"$if(si(darkmode)=1, #FF1B1B1D, #FFFFFFFF)$"}}
  ```

## 8. 与本项目的对照（我们的实现）

| Kustom 公式编辑器 | 本项目 `src/overlay` |
|---|---|
| 属性三态 → 唤起 Kode 编辑器 | ✅ `PropertyEditorPanel` 字段三态条（值 / 公式 / 变量） |
| `FormulaTextField` + `AnnotatedFormula` 语法高亮输入 | ⚠️ 有公式文本框 + `FunctionCatalog`，**缺语法高亮**（可评估） |
| 函数目录插入（带占位模板） | ✅ `FunctionCatalog.cs` 列 19 函数 + 插入模板 |
| `PreviewGlobalDialog` 选全局变量 | ✅ `GvManagerPanel` + `GvStore`（类型 NUMBER/TEXT/COLOR/FONT，**缺 LIST**） |
| 实时预览 `PresetPreview` | ✅ `DirtyScheduler` 实时重算 |
| 校验 `ErrorSnackBar` / `ErrorMessage` | ⚠️ 公式求值 `try/catch` 容错，但**无内联错误态 UI**（可评估） |
| `EDIT_KODE` 意图 / 多 Kode 编辑器 | ➖ 桌面端单进程内联面板，无需意图分发 |
| 暗色模式 `si(darkmode)` 配色指引 | ✅ 有暗色主题，可在配色指引复用 |

**结论**：本项目「属性三态 + 函数目录 + 变量管理器 + 实时重算」方向正确；若要做得更精，
可补 **① 公式语法高亮** 与 **② 公式内联错误提示** 两项（对应 Kustom 的
`AnnotatedFormula` + `InputTextFieldError`），全局变量类型可补 `LIST`。
